## 무엇이 "패키징" 인가

지금까지의 `npm start` 는 우리 컴퓨터에서만 동작해요. 사용자에게 보낼 땐 OS 가 알아보는 형태 — 맥은 `.dmg`/`.app`, 윈도우는 `.exe` 설치 파일, 리눅스는 `.deb`/`.rpm`/`.AppImage` 가 필요해요. **Electron Forge** 가 이걸 한 번에 해 줍니다.

<EasyNote>
"내 도시락(=실행 코드)을 친구에게 보내려면 도시락통(=설치 파일)에 담아야 한다"가 정확한 비유예요.
</EasyNote>

## Electron Forge 도입

<Steps>

### 설치 + 초기화

```bash filename="terminal"
npm install --save-dev @electron-forge/cli
npx electron-forge import
```

`import` 명령은 기존 프로젝트의 `package.json` 을 Forge 가 쓰는 형태로 자동 변환해 줍니다.

### 패키지 만들기 (현재 OS 용)

```bash filename="terminal"
npm run make
```

`out/` 폴더에 OS 별 설치 파일이 생깁니다.

</Steps>

## 자주 꼬이는 부분

- **아이콘** — 256×256 PNG + 맥은 `.icns`, 윈도우는 `.ico` 도 필요
- **앱 이름·버전** — `package.json` 의 `productName` 과 `version` 이 그대로 사용됨
- **빌드 시간** — 첫 빌드는 5~10분 걸릴 수 있음 (네이티브 의존성 다운로드)

<Callout type="warning">
크로스 OS 빌드(맥에서 윈도우 .exe 만들기 같은)는 _이론상_ 가능하지만 코드 서명 때문에 실무에선 추천하지 않아요. **각 OS 의 가상머신/러너에서 빌드** 하는 게 정석입니다.
</Callout>

## 쉽게 풀어 설명

지금까지의 우리 앱은 _개발자 컴퓨터 안에서만_ 사는 새끼 새 같아요. 친구한테 보여주려면 둥지 밖으로 꺼내 — _OS 가 알아보는 모양_ 으로 바꿔 줘야 해요. 맥은 `.dmg` 디스크 이미지, 윈도우는 `.exe` 설치 마법사, 리눅스는 `.deb`/`.rpm`/`.AppImage` 등 배포판별 포맷이 있어요. **Electron Forge** 가 우리 코드를 이 세 모양으로 한 번에 포장해 주는 _자동 도시락 기계_ 라고 생각하면 됩니다.

어렵게 느껴지는 부분은 _아이콘 파일을 OS 별로 따로 준비해야 한다_ 는 점, 그리고 _첫 빌드는 5~10분쯤 걸린다_ 는 점이에요. 처음엔 "왜 이렇게 오래?" 싶지만, Forge 가 백그라운드에서 Chromium 바이너리·OS별 도구체인·아이콘 변환 도구 같은 큰 부품들을 받기 때문이에요. 한 번 받아두면 다음부터는 1~2분이면 끝.

가장 자주 혼동하는 게 _크로스 OS 빌드_ 예요. "맥에서 윈도우 .exe 도 만들 수 있나?" 이론상 가능하지만 _코드 서명_ 이 막혀 실무에선 안 통해요. 그래서 회사들은 _OS 별 가상머신/CI 러너_ 에서 따로 빌드해요. 다음 [배포·자동 업데이트](/ko/docs/tutorial/publishing) 페이지에서 이 도시락을 _친구 손에 어떻게 전달하나_ 를 다룹니다.
