## 한 줄 요약

Electron + Webpack/Vite + maker(.dmg/.exe/.deb 만드는 도구) + publisher(GitHub/S3) 를 _하나의 명령어 체인_ 으로 묶어 둔 도구가 Forge 예요.

| 단계 | 명령어 |
|---|---|
| 개발 모드 | `npm start` |
| 패키지(아직 설치 파일 X) | `npm run package` |
| 설치 파일 만들기 | `npm run make` |
| 업로드까지 | `npm run publish` |

## 새 프로젝트 시작

```bash filename="terminal"
npx create-electron-app@latest my-app --template=vite-typescript
cd my-app
npm start
```

<EasyNote>
`--template=` 한 줄이 _빈 종이 대신 격자 노트로 시작_ 하게 해 줍니다. Vite 의 빠른 HMR 까지 자동.
</EasyNote>

## 구성 파일

```js filename="forge.config.js"
module.exports = {
  packagerConfig: { asar: true, icon: 'assets/icon' },
  makers: [
    { name: '@electron-forge/maker-squirrel' },   // Windows
    { name: '@electron-forge/maker-zip',  platforms: ['darwin'] },
    { name: '@electron-forge/maker-deb' },         // Debian/Ubuntu
    { name: '@electron-forge/maker-rpm' },         // Fedora
  ],
}
```

## 쉽게 풀어 설명

옛날엔 Electron 앱을 배포하려면 _도구를 직접 다 모아_ 와야 했어요. 빌드 도구, 인스톨러 만드는 도구, 코드 서명 도구, 업로드 도구. 도구마다 설정 파일·옵션이 달라서 첫 배포에만 몇 주가 걸리곤 했죠. **Electron Forge** 는 _그 도구 세트를 하나의 박스에 모아 둔 도구함_ 이에요. `npm start` 한 줄이 개발, `npm run make` 한 줄이 설치 파일, `npm run publish` 한 줄이 업로드까지 끝내요.

가장 큰 강점은 _템플릿_ 이에요. `npx create-electron-app@latest my-app --template=vite-typescript` 한 줄이면, Vite + TypeScript + 최신 보안 옵션이 모두 켜진 _즉시 시작 가능한 프로젝트_ 가 생겨요. 빈 종이 대신 _격자가 그려진 종이_ 부터 시작하는 셈이에요.

설정 파일은 `forge.config.js` 한 곳에서 모두 관리해요. _packagerConfig_ 는 앱 자체의 패키징 옵션(아이콘, asar 등), _makers_ 는 어떤 설치 파일을 만들지 (Windows .exe, macOS .zip/.dmg, Linux .deb/.rpm), _publishers_ 는 어디에 올릴지(GitHub, S3 등). 이 세 블록만 손에 익으면 배포 자동화의 거의 모든 게 가능합니다.
