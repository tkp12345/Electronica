## Snap 의 특징

- **Universal Linux** — 배포판에 상관없이 같은 `.snap` 파일
- **자동 업데이트** — Snap Store 가 백그라운드에서 알아서 갱신
- **샌드박스(AppArmor 기반)** — 권한이 명시적
- **Stable / Candidate / Beta / Edge** 4채널 관리

## 업로드

```bash filename="terminal"
snapcraft login
snapcraft upload my-app_1.0.0_amd64.snap --release=stable
```

## 다른 옵션과 비교

- `.deb` / `.rpm` — 가벼움, 배포판 특화
- `.AppImage` — 설치 불필요, 단일 파일
- **Snap** — 자동 업데이트가 큰 장점

## 쉽게 풀어 설명

리눅스는 _배포판이 너무 많다_ 는 게 강점이자 약점이에요. 우분투, 페도라, 데비안, 아치 등 각 배포판마다 _좋아하는 패키지 포맷_ 이 달라요. Snap 은 그 차이를 _통째로 가린_ 컨테이너형 포맷이에요. 한 번 `.snap` 으로 만들면 _배포판이 무엇이든_ 같은 방식으로 설치되고, _자동 업데이트_ 까지 OS 가 알아서 처리해 줘요. 사용자 입장에서는 "리눅스에서도 앱이 알아서 새 버전으로 바뀌는" 익숙한 경험이 됩니다.

다른 옵션과 비교하면 _크기-편의성-자동화_ 의 삼각관계가 보여요. `.deb`/`.rpm` 은 _가장 가벼우면서 배포판 특화_ 라 네이티브 느낌이 좋지만, 자동 업데이트는 우리가 직접 구현해야 해요. `.AppImage` 는 _설치도 안 하고 더블클릭만으로_ 도는 단일 파일이라 USB 로 옮기기 좋지만, 자동 업데이트가 약점. Snap 은 _컨테이너라서 약간 무거운_ 대신 모든 게 자동.

채널 운영 팁: Snap Store 는 _Stable / Candidate / Beta / Edge_ 네 채널을 동시에 굴려요. 새 기능은 Edge 에 먼저 올려서 _얼리어답터_ 에게만 풀고, 안정 확인 후 Stable 로 승급. 이 단계적 롤아웃이 큰 사고를 막아 줘요.
