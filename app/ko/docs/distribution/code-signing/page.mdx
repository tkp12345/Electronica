## 왜 필요한가

- **macOS**: 서명 안 되면 Gatekeeper 가 _"손상되었거나 신뢰할 수 없는 앱"_ 으로 막음
- **Windows**: SmartScreen 이 _"인식되지 않은 앱"_ 빨간 경고 표시
- **자동 업데이트**: 새 버전이 같은 서명을 갖고 있어야 적용됨

<EasyNote>
"이 도시락은 우리 가게에서 만든 거예요" 를 _봉인 스티커_ 로 증명하는 거예요. 봉인 없는 도시락은 손님이 안 사 먹죠.
</EasyNote>

## macOS 서명 + 공증(notarization)

```js filename="forge.config.js"
module.exports = {
  packagerConfig: {
    osxSign: { identity: 'Developer ID Application: Your Name (TEAMID)' },
    osxNotarize: {
      tool: 'notarytool',
      appleId: process.env.APPLE_ID,
      appleIdPassword: process.env.APPLE_PASSWORD,
      teamId: process.env.APPLE_TEAM_ID,
    },
  },
}
```

공증(notarization)은 _애플 서버가 앱을 검사한 뒤 "안전함" 도장을 찍는_ 과정이에요. 서명만으론 부족, 공증까지 마쳐야 macOS Catalina+ 에서 경고 없이 열림.

## Windows 서명

EV 인증서(파일 또는 USB 토큰)로 `signtool` 을 통해 `.exe` 와 인스톨러에 서명해요. Forge 의 `@electron-forge/maker-squirrel` 옵션에 `certificateFile` 을 지정합니다.

```js filename="forge.config.js"
module.exports = {
  makers: [
    {
      name: '@electron-forge/maker-squirrel',
      config: {
        certificateFile: process.env.WINDOWS_CERT_FILE, // .pfx 경로
        certificatePassword: process.env.WINDOWS_CERT_PASSWORD,
      },
    },
  ],
}
```

내부적으로는 Squirrel.Windows 가 `signtool` 을 대신 실행해 줘요. 서명된 `.exe` 는 `%LOCALAPPDATA%\<App>` 에 사용자별로 설치되며 관리자 권한 프롬프트가 필요 없습니다.

## CI 에서 안전하게

- 인증서는 **비밀 환경변수**(GitHub Secrets, 1Password Service Account 등)
- 빌드 로그에 인증서 경로/패스워드 노출 금지
- 인증서 만료일을 캘린더에 등록

## 쉽게 풀어 설명

택배 상자에 _봉인 스티커_ 가 붙어 있지 않으면 사람들은 받기를 꺼려해요. 누가 중간에 열어 봤을지 모르니까요. 디지털 앱도 똑같아요. _코드 서명_ 은 "이 앱은 우리(아무개 회사) 가 만들었고, 그 후로 누구도 손대지 않았다" 라는 봉인 스티커예요. 맥과 윈도우는 봉인 없는 앱을 사용자에게 _경고와 함께_ 보여주거나 아예 _차단_ 해 버려요.

맥은 _서명 + 공증(notarization)_ 두 단계가 필요해요. 서명만 하면 우리 컴퓨터에선 잘 열리지만, _다른 사용자의 맥은 안 열어_ 줘요. 공증은 _애플 서버가 우리 앱을 검사한 뒤 "안전함" 도장을 추가로 찍어 주는_ 과정이에요. 이 도장이 있어야 맥OS Catalina 이후 버전에서 경고 없이 열려요. 윈도우는 EV 인증서로 `.exe` 와 설치 파일에 서명하는 게 표준이에요.

CI 에서 다룰 땐 _인증서 자체가 가장 민감한 비밀_ 이에요. 절대로 GitHub 저장소에 커밋하지 마세요. 환경변수나 시크릿 매니저에 보관하고, 빌드 로그에 경로·비밀번호가 노출되지 않도록 가려야 해요. 마지막으로 _만료일을 캘린더에 등록_ — 인증서가 갑자기 만료되면 자동 업데이트까지 같이 멈춥니다.
