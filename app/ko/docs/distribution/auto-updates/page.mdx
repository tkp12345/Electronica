## 가장 쉬운 방법

```js filename="main.js"
const { updateElectronApp } = require('update-electron-app')
updateElectronApp({ repo: 'your-name/your-app' })
```

한 줄로 _"앱 켤 때마다 → GitHub Releases 확인 → 새 버전 있으면 다운로드 → 재시작 시 적용"_ 까지 끝납니다.

<EasyNote>
"브라우저가 알아서 새 버전 받아 두고, 다음에 켤 때 갈아끼우는" 그 동작이에요. 사용자가 _버튼을 누르지 않아도_ 동작합니다.
</EasyNote>

## 내부 동작

1. **autoUpdater** (Electron 내장) 가 _업데이트 서버_ 에 _현재 버전_ 보내고 새 버전 있는지 묻는다
2. 새 버전이 있으면 백그라운드 다운로드
3. 다운로드 끝나면 `update-downloaded` 이벤트
4. 다음 종료/재시작 시 _원자적_ 으로 교체

## 코드 서명 없으면 안 됨

새 버전이 _같은 서명_ 을 갖고 있어야 OS 가 받아들여요. 미서명 빌드는 자동 업데이트 자체가 작동하지 않습니다.

<Callout type="info">
플랫폼마다 동작이 조금 달라요. _(macOS)_ 새 `.app` 이 실행 중인 앱과 **같은 Team ID / Developer ID** 로 서명돼 있어야 합니다 — Squirrel.Mac 이 이를 검증하고, 서명이 다르면 교체를 거부해요. _(Windows)_ Squirrel.Windows 는 새 `.exe` 를 설치하면서 `%LOCALAPPDATA%\<App>` 아래 사용자별 폴더를 갱신합니다. `signtool` 을 통한 Authenticode 서명은 SmartScreen 경고를 막아 주지만, 교체 자체에 서명 일치가 엄격히 강제되지는 않아요.
</Callout>

## 자체 서버를 쓰고 싶다면

```js
const { autoUpdater } = require('electron')

autoUpdater.setFeedURL({
  url: 'https://updates.example.com/win32/' + app.getVersion(),
})

autoUpdater.checkForUpdates()
```

서버는 _Squirrel.Mac / Squirrel.Windows / Nuts / Hazel_ 같은 오픈소스를 직접 운영하거나 자체 구현.

## 쉽게 풀어 설명

좋은 앱과 _진짜로 사용자 곁에 오래 머무는_ 앱의 차이는 _업데이트_ 에 있어요. 매번 사용자에게 "새 버전 다운받으세요!" 메일을 보낼 순 없으니, 앱이 _스스로_ 새 버전을 확인하고 받아 적용해야 해요. Electron 의 `update-electron-app` 패키지가 그 일을 _한 줄_ 로 해 줍니다. `updateElectronApp({ repo: 'your/repo' })`.

이 한 줄 뒤에서 일어나는 일이 꽤 똑똑해요. 앱이 켜질 때 GitHub Releases 에 _현재 버전과 가장 최신 버전을 비교_, 새 버전이 있으면 _백그라운드에서 다운로드_, 다운로드 완료 후 _사용자가 다음에 종료할 때 조용히 적용_. 사용자는 어느 날 켰을 때 _"어, 이거 디자인이 살짝 바뀌었네?"_ 만 느껴요.

자체 서버를 쓰고 싶다면 `autoUpdater.setFeedURL` 로 _업데이트 피드 URL_ 만 알려 주면 돼요. 서버 쪽은 Squirrel.Mac/Windows 프로토콜을 따르거나 Hazel/Nuts 같은 오픈소스를 쓰면 빠르게 만들 수 있어요. 가장 중요한 건 _코드 서명이 새 버전과 옛 버전에 동일하게 유지_ 되어야 한다는 점. 서명이 어긋나면 OS 가 "이건 다른 앱이야" 라고 판단해 업데이트가 안 돼요.
