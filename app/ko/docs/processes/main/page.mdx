## 정체

Node.js 프로그램 그 자체입니다. `package.json` 의 `main` 필드가 가리키는 파일이 시작점이에요.

```js filename="main.js"
const { app, BrowserWindow } = require('electron')

app.whenReady().then(() => {
  new BrowserWindow().loadURL('https://example.com')
})
```

## 할 수 있는 일

- 창(`BrowserWindow`)을 만들고/닫기
- 메뉴/트레이/대화상자/단축키
- Node 표준 라이브러리 전부 (`fs`, `path`, `child_process`, `net` ...)
- npm 패키지 자유롭게 (`axios`, `better-sqlite3` 등)
- IPC 채널 핸들러 등록 (`ipcMain.handle`)

## 할 수 없는 일

- DOM 을 직접 만지기 — Main 에는 `document` 가 없음
- 화면을 직접 그리기 — 그건 Renderer 의 영역

<Callout type="info" emoji="🧒">
Main 안에서 `document.getElementById(...)` 같은 걸 쓰면 곧장 에러예요. _감독이 무대 위로 뛰어 올라가 연기하려는 격_.
</Callout>

## 라이프사이클 이벤트 (자주 쓰는 것)

| 이벤트 | 언제 |
|---|---|
| `ready` (= `app.whenReady()`) | 모든 모듈 로드 끝, 창을 만들 수 있는 시점 |
| `window-all-closed` | 마지막 창이 닫혔을 때 — 보통 여기서 `app.quit()` |
| `activate` (macOS) | dock 아이콘 클릭으로 앱이 다시 활성화 |
| `before-quit` | 종료 직전 — 저장 안 한 데이터 처리에 유용 |
| `will-quit` | 종료 직전의 마지막 단계 — 단축키 해제 등 |
