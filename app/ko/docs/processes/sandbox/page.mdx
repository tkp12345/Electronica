## 한마디로

Sandbox 가 켜진 Renderer 는 _OS 의 가장 낮은 권한_ 으로 돌아갑니다. 파일 시스템·네트워크 같은 OS API 를 직접 못 부르고, 오직 IPC 로 Main 에 _부탁_ 만 할 수 있어요.

<EasyNote>
"손님은 카운터 너머로 손을 못 뻗어요. 메뉴판 보고 종업원(IPC)에게 _시키기만_ 가능."
</EasyNote>

## 기본값은 ON

Electron 20+ 부터 모든 BrowserWindow 의 `sandbox` 가 기본 true 입니다. 굳이 끄지 마세요.

```js filename="main.js"
new BrowserWindow({
  webPreferences: {
    sandbox: true, // 기본값 — 명시해도 좋다
    contextIsolation: true,
  },
})
```

## 가끔 끄고 싶어지는 이유 (대부분 함정)

- "preload 에서 Node API 쓰고 싶어서" → 잘못된 이유 (preload 는 sandbox 켜진 상태에서도 일부 Node 모듈을 쓸 수 있게 따로 설계됨)
- "오래된 라이브러리가 require 호출함" → 그 라이브러리를 Main 쪽으로 옮기는 게 정답
