## Renderer 의 두 이벤트

```js filename="renderer.js"
window.addEventListener('online', () => setBanner('연결됨'))
window.addEventListener('offline', () => setBanner('오프라인'))
```

## Main 에서 감지하기 (창 없이도)

```js filename="main.js"
const { BrowserWindow, app } = require('electron')

app.whenReady().then(() => {
  const hidden = new BrowserWindow({ show: false })
  hidden.webContents.executeJavaScript(`
    window.addEventListener('online', () => { /* IPC 로 알림 */ })
  `)
})
```

<EasyNote>
Main 자체에는 `navigator` 가 없어서, _숨겨진 창 하나_ 를 만들어 거기서 이벤트를 듣고 IPC 로 받아 오는 패턴을 많이 써요.
</EasyNote>

## 진짜로 살아있는지 확인하려면 핑

`navigator.onLine` 은 Wi-Fi 연결만 확인. 서버 헬스체크는 별도로:

```js
async function isAlive() {
  try {
    const r = await fetch('/api/ping', { cache: 'no-store' })
    return r.ok
  } catch {
    return false
  }
}
```

## 쉽게 풀어 설명

네트워크는 _전등 스위치_ 처럼 깔끔하게 켜졌다 꺼졌다 하지 않아요. 와이파이는 신호가 약해서 1초마다 끊기고 붙기를 반복할 수도 있고, 와이파이는 연결돼 있는데 _우리 서버_ 만 응답이 없을 수도 있어요. 그래서 두 단계의 감지가 필요해요.

첫째는 _OS 가 알려 주는 신호_. 브라우저의 `window.addEventListener('online' / 'offline', ...)` 이벤트가 그거예요. 가장 빠르고 가장 가벼워서 _UI 의 첫 반응_ 에 쓰기 좋아요. 둘째는 _진짜 살아있는지 직접 묻는 핑(ping)_. 우리 서버에 1초마다 가벼운 요청을 보내 응답을 보면, "와이파이는 있는데 서버가 죽음" 같은 상황까지 잡아낼 수 있어요.

Main 프로세스 자체에는 `navigator` 가 없어요. 그래서 Main 에서 온오프를 알고 싶다면 _숨겨진 작은 창_ 하나를 띄워서 그 안에서 `online`/`offline` 이벤트를 듣고 IPC 로 받는 패턴을 흔히 써요. 약간 어색해 보이지만, 실무에서 가장 안정적인 방법이에요.
