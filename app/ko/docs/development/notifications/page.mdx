## Renderer 에서

```js
new Notification('빌드 완료', {
  body: '5.2초 걸렸어요',
  silent: false,
})
```

## Main 에서 (창 없이도 가능)

```js filename="main.js"
const { Notification } = require('electron')

new Notification({
  title: '업데이트 준비됨',
  body: '재시작 시 적용됩니다',
}).show()
```

<EasyNote>
같은 `Notification` 이름이지만 _맥은 위 오른쪽_, _윈도우는 아래 오른쪽_, _리눅스는 데스크톱마다 다름_ — OS 알림 센터의 모양은 OS 가 결정해요. 우리는 신호만 보내는 거예요.
</EasyNote>

## 클릭/응답 처리

```js filename="main.js"
const n = new Notification({ title: 'Hi', body: 'Click me' })
n.on('click', () => focusMainWindow())
n.show()
```

## 자주 막히는 부분

- macOS: 알림이 안 뜨면 _시스템 환경설정 → 알림_ 에서 앱 권한 확인
- Windows: AppUserModelID 가 잘못되면 알림이 끊김 (코드 서명 + 패키지 후에만 정상)

## 쉽게 풀어 설명

웹페이지의 `new Notification('...')` 코드를 그대로 데스크톱 앱에서 쓸 수 있어요. 모양은 똑같지만 _도착하는 위치_ 가 달라요. 브라우저였다면 페이지 안에 작은 풍선이 떴겠지만, Electron 안에서는 _OS 의 진짜 알림 센터_ — 맥은 우측 상단, 윈도우는 우측 하단 — 로 흘러갑니다. 즉 사용자는 우리 앱이 꺼져 있어도 그 알림을 볼 수 있어요.

같은 API 이지만 OS 마다 _보이는 모양은 우리가 결정 못 해요_. 둥근 모서리·아이콘 배치·재생 사운드는 OS 가 정해 줍니다. 우리는 _신호_ 만 보내는 거예요. 그게 오히려 좋아요. 사용자가 다른 앱에서 받던 알림과 _똑같은 톤_ 으로 우리 알림도 도착하니까요.

가장 자주 막히는 함정 두 가지: ① 맥에서 처음 알림을 보내면 _권한 요청_ 이 뜨는데, 거절당하면 그 뒤로 영원히 안 떠요. 시스템 환경설정 → 알림 에서 다시 허용해야 해요. ② 윈도우에서는 _AppUserModelID_ 가 잘못 설정되면 알림이 통째로 안 와요. 이 ID 는 _패키징·코드 서명_ 을 마친 빌드에서만 정상 동작합니다.
