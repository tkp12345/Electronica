## 추천 3종

| 이름 | 특징 |
|---|---|
| **electron-quick-start** | 가장 작은 시작점, 패턴 익히기 좋음 |
| **electron-forge** + 템플릿 (`vite`, `webpack-typescript`) | 공식 풀스택 도구체인 |
| **electron-react-boilerplate** | React + TypeScript + Redux/RTK + Jest 까지 세팅 |

## Forge 템플릿으로 시작

```bash filename="terminal"
npx create-electron-app@latest my-app --template=vite-typescript
```

<EasyNote>
`--template=` 한 줄이 _"빈 종이 대신 격자 노트로 시작"_ 이라는 뜻이에요. Vite 의 빠른 HMR 까지 자동 세팅.
</EasyNote>

## 보일러플레이트를 고를 때 기준

- _이 회사 코드에 들어갈 건가?_ → React 보일러플레이트
- _내 학습용?_ → quick-start
- _데모/실험?_ → Fiddle

## 쉽게 풀어 설명

새 집을 짓는다고 _땅에 직접 벽돌을 쌓을 필요_ 는 없어요. 누군가 미리 _기초·골조·전기 배선_ 까지 세워 둔 _샘플 하우스_ 가 있으면 거기서 시작해 인테리어만 우리 취향대로 바꾸면 되죠. 보일러플레이트가 바로 그 샘플 하우스예요.

세 가지 옵션의 _크기 차이_ 만 기억하면 돼요. **electron-quick-start** 는 _1평짜리 원룸_ — 가장 작아서 패턴 학습용으로 최고예요. 어디서부터가 우리 코드인지 한눈에 보여요. **Electron Forge 템플릿** 은 _3룸 아파트_ — Webpack/Vite 가 깔려 있어 실제로 무언가를 만들 때 부족함이 없어요. **electron-react-boilerplate** 는 _제대로 갖춰진 빌라_ — React, TS, Redux, Jest 까지 미리 깔려 있어 회사용 시작점에 적합해요.

가장 흔한 실수는 _학습 단계에서 너무 큰 보일러플레이트_ 를 고르는 거예요. 파일이 100개쯤 깔리면 "어디부터 봐야 하지?" 가 되어 도리어 학습이 느려져요. _가장 작게 시작해서 → 필요해질 때 부품을 더하는_ 방향이 정답입니다.
