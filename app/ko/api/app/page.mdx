`app` 모듈은 **Main 프로세스** 에 살아요. 프로그램이 언제 시작하는지, 언제 종료해야 하는지, 파일이 어디에 들어가야 하는지, OS 와 어떻게 연결되는지를 모두 아는 단 하나의 객체예요.

```js filename="main.js"
const { app } = require('electron')

app.whenReady().then(() => {
  // 앱이 여기서 시작돼요
})
```

## 생명주기, 단계별로

`app` 의 대부분은 _타이밍_ 에 관한 거예요 — 알맞은 순간에 알맞은 일을 하는 것. 아래 단계를 눌러 보며 부팅부터 종료까지의 전체 여정을 따라가 보세요.

<AppLifecycleTimeline lang="ko" />

<EasyNote>
`app` 은 프로그램의 **무대 감독** 이에요. 화면에 픽셀 하나 직접 그리지 않지만, 막을 언제 올릴지(`ready`), 배우가 언제 퇴장할지(`window-all-closed`), 언제 조명을 끌지(`will-quit`)를 결정해요.
</EasyNote>

## 이벤트(Events)

`app.on('이벤트명', 핸들러)` 로 듣습니다. _(macOS)_ / _(Windows)_ / _(Linux)_ 표시는 해당 플랫폼에서만 발생해요.

### 생명주기

| 이벤트 | 발생 시점 |
|---|---|
| `will-finish-launching` | 부팅을 막 끝내려는 시점 (초기 리스너 설정) |
| `ready` | 초기화 완료 — 창을 만들어도 됨 (macOS 는 `launchInfo` 전달) |
| `window-all-closed` | 마지막 창이 닫힘 |
| `before-quit` | 종료 시작 — `preventDefault()` 로 취소 가능 |
| `will-quit` | 모든 창이 닫혔고 곧 종료 — 마지막 정리 |
| `quit` | 앱이 종료되는 중 (`exitCode` 전달) |

```js filename="lifecycle.js"
app.whenReady().then(createWindow)

app.on('window-all-closed', () => {
  // macOS 앱은 보통 창이 없어도 살아 있어요
  if (process.platform !== 'darwin') app.quit()
})

app.on('before-quit', (event) => {
  if (hasUnsavedWork()) {
    event.preventDefault() // 종료를 막아요
    showSaveDialog()
  }
})
```

<Callout type="warning">
`ready`/`whenReady()` 이전에는 절대 `BrowserWindow` 를 만들지 마세요. GUI 계층이 아직 초기화되지 않아 앱이 죽어요.
</Callout>

### macOS 전용

| 이벤트 | 발생 시점 |
|---|---|
| `open-file` _(macOS)_ | 사용자가 앱으로 파일을 열려고 함 |
| `open-url` _(macOS)_ | 앱에 등록된 URL 을 엶 |
| `activate` _(macOS)_ | 앱이 활성화됨 (예: Dock 아이콘 클릭) |
| `did-become-active` _(macOS)_ | 앱이 활성 상태가 됨 |
| `did-resign-active` _(macOS)_ | 앱이 포커스를 잃음 |
| `new-window-for-tab` _(macOS)_ | 네이티브 "새 탭" 버튼 클릭 |
| `continue-activity` _(macOS)_ | 다른 기기의 Handoff 활동을 여기서 이어받으려 함 |
| `will-continue-activity` _(macOS)_ | Handoff 활동을 이어받기 직전 |
| `continue-activity-error` _(macOS)_ | Handoff 활동 이어받기 실패 |
| `activity-was-continued` _(macOS)_ | 이 기기 활동이 다른 기기에서 이어짐 |
| `update-activity-state` _(macOS)_ | Handoff 활동이 다른 기기에서 재개되려 함 |

```js filename="macos.js"
// macOS: Dock 아이콘을 누르면 창을 다시 만들어요
app.on('activate', () => {
  if (BrowserWindow.getAllWindows().length === 0) createWindow()
})

// macOS: myapp:// 같은 커스텀 프로토콜 처리
app.on('open-url', (event, url) => {
  event.preventDefault()
  handleDeepLink(url)
})
```

### 창 & WebContents

| 이벤트 | 발생 시점 |
|---|---|
| `browser-window-created` | 새 `BrowserWindow` 생성됨 |
| `browser-window-focus` | 창이 포커스를 얻음 |
| `browser-window-blur` | 창이 포커스를 잃음 |
| `web-contents-created` | 새 `WebContents` 생성됨 (모든 페이지/webview) |
| `render-process-gone` | 렌더러 프로세스가 비정상 종료됨 (크래시/강제종료) |
| `child-process-gone` | 렌더러가 아닌 자식 프로세스가 사라짐 |

```js filename="harden.js"
// 생성되는 모든 페이지를 안전하게 잠가요 — 보안 베스트 프랙티스
app.on('web-contents-created', (event, contents) => {
  contents.setWindowOpenHandler(() => ({ action: 'deny' }))
})

app.on('render-process-gone', (event, contents, details) => {
  // details.reason: 'crashed' | 'killed' | 'oom' | ...
  reloadOrNotify(contents, details.reason)
})
```

### 보안 & 시스템

| 이벤트 | 발생 시점 |
|---|---|
| `certificate-error` | URL 의 TLS 인증서 검증 실패 |
| `select-client-certificate` | 클라이언트 인증서 요청됨 |
| `login` | `WebContents`/유틸리티 프로세스가 기본·프록시 인증 필요 |
| `gpu-info-update` | GPU 정보가 갱신됨 |
| `accessibility-support-changed` _(macOS/Windows)_ | OS 접근성 지원이 토글됨 |
| `session-created` | 새 `Session` 이 생성됨 |
| `second-instance` | 단일 인스턴스 잠금을 쥔 상태에서 또 다른 인스턴스가 실행됨 |

```js filename="single-instance.js"
const gotLock = app.requestSingleInstanceLock()

if (!gotLock) {
  app.quit() // 이미 실행 중인 복사본이 있으니 이건 종료
} else {
  app.on('second-instance', (event, argv, workingDir, extraData) => {
    // 두 번째 복사본 실행 시도 → 새로 열지 않고 기존 창을 앞으로
    if (mainWindow) {
      if (mainWindow.isMinimized()) mainWindow.restore()
      mainWindow.focus()
    }
  })
}
```

<EasyNote>
`second-instance` 가 바로 "한 창만 뜨는" 앱의 원리예요. 아이콘을 또 눌러도 새 복사본을 띄우는 대신 이미 떠 있는 창을 앞으로 가져와요.
</EasyNote>

## 메서드(Methods)

### 생명주기 & 종료

| 메서드 | 하는 일 |
|---|---|
| `app.quit()` | 모든 창을 닫고 종료 시도 (`before-quit` 으로 취소 가능) |
| `app.exit([exitCode])` | 정리 없이 **즉시** 종료, exitCode 와 함께 |
| `app.relaunch([options])` | 현재 인스턴스 종료 후 앱 재시작 |
| `app.isReady()` | 초기화가 끝났으면 `true` |
| `app.whenReady()` | 준비 완료 시 resolve 되는 `Promise` |

```js filename="relaunch.js"

app.relaunch()
app.exit(0)
```

## 쉽게 풀어 설명

`app` 모듈은 **프로그램 전체의 관리자** 예요. 창도 아니고 페이지도 아닌, 그 모든 것 위에서 돌아가는 존재죠.

크게 세 가지 일이 거의 전부예요:

1. **타이밍.** "언제 창을 만들 수 있지?" → `ready` 를 기다려요. "마지막 창이 닫히면 뭘 하지?" → `window-all-closed` 를 처리해요. "종료 전에 저장할 게 있나?" → `before-quit` 을 써요. 위의 인터랙티브 타임라인이 바로 이 일이에요.
2. **장소.** "내 설정은 어디 저장하지?" → `getPath('userData')`. "지금 빌드된 버전인가?" → `isPackaged`. OS 는 용도별 폴더가 정해져 있고, `app` 이 매번 알맞은 경로를 건네줘요.
3. **OS 연동.** 딥링크(`setAsDefaultProtocolClient`), 한 복사본만 실행(`requestSingleInstanceLock` + `second-instance`), 로그인 시 실행, Dock 배지, 점프 리스트 — 웹앱을 네이티브 앱처럼 느끼게 하는 작은 손길들이에요.

세 이름만 외우면 90%는 끝나요: **`whenReady()`**(여기서 시작), **`window-all-closed`**(언제 종료할지), **`getPath('userData')`**(데이터가 사는 곳). 나머지는 찾아보면 돼요 — 이 페이지가 바로 그 용도예요.
