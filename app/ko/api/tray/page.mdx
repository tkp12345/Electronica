`Tray` 는 **Main 프로세스** 에 살아요. 운영체제의 상태 영역 — macOS 의 메뉴 막대, Windows 의 알림 영역(오른쪽 아래), Linux 의 상태 트레이 — 에 앱 아이콘을 올려서, 창이 하나도 열려 있지 않아도 사람들이 앱에 닿을 수 있게 해줘요.

대부분의 API 모듈과 달리 `Tray` 는 **클래스** 예요. `new Tray(image)` 로 인스턴스를 만들고, 그 인스턴스의 메서드를 호출하는 방식이에요.

```js filename="main.js"
const { app, Tray, Menu, nativeImage } = require('electron')

let tray // 가비지 컬렉션되지 않도록 참조를 유지해요

app.whenReady().then(() => {
  const icon = nativeImage.createFromPath('/path/to/icon.png')
  tray = new Tray(icon)
  tray.setToolTip('My App')

  const menu = Menu.buildFromTemplate([
    { label: '열기', click: () => mainWindow.show() },
    { label: '종료', role: 'quit' },
  ])
  tray.setContextMenu(menu)
})
```

<Callout type="warning">
`Tray` 인스턴스는 반드시 오래 사는 변수(예: 모듈 레벨의 `let tray`)에 저장하세요. 지역 변수에만 담아 두면 가비지 컬렉션되어 아이콘이 소리 없이 사라질 수 있어요.
</Callout>

## 생성자(Constructor)

```js filename="constructor.js"
// image: NativeImage 객체 또는 경로 문자열
// guid:  선택 사항, Windows/macOS — 재시작해도 OS 가 아이콘 위치를
//        기억하도록 하는 안정적인 UUID
const tray = new Tray(image, guid)
```

| 인자 | 타입 | 무엇인가 |
|---|---|---|
| `image` | `NativeImage \| string` | 표시할 아이콘 (객체 또는 파일 경로) |
| `guid` _(Windows/macOS)_ | `string` (선택) | 앱을 재시작해도 아이콘 위치를 유지시키는 UUID |

<EasyNote>
macOS 에서는 **템플릿 이미지**(`…Template.png` 라는 이름의 흑백 PNG)를 쓰세요. 그러면 macOS 가 자동으로 색을 입혀서, 밝은 메뉴 막대와 어두운 메뉴 막대 모두에서 자연스럽게 보여요.
</EasyNote>

## 인스턴스 이벤트(Instance Events)

`tray.on('이벤트명', 핸들러)` 로 듣습니다. _(macOS)_ / _(Windows)_ / _(Linux)_ 표시는 해당 플랫폼에서만 발생해요.

### 클릭

| 이벤트 | 발생 시점 |
|---|---|
| `click` | 트레이 아이콘이 클릭됨 (Linux 에서는 반드시 좌클릭이 아니라 "활성화"를 의미) |
| `right-click` _(macOS/Windows)_ | 트레이 아이콘이 우클릭됨 |
| `double-click` _(macOS/Windows)_ | 트레이 아이콘이 더블클릭됨 |
| `middle-click` _(Windows)_ | 트레이 아이콘이 가운데 버튼으로 클릭됨 |

```js filename="clicks.js"
// 아이콘을 클릭하면 메인 창을 토글해요
tray.on('click', () => {
  mainWindow.isVisible() ? mainWindow.hide() : mainWindow.show()
})

// Windows/macOS 에서 우클릭은 보통 "메뉴 띄우기" 제스처예요
tray.on('right-click', () => {
  tray.popUpContextMenu()
})
```

### 마우스 이동

| 이벤트 | 발생 시점 |
|---|---|
| `mouse-down` _(macOS)_ | 아이콘 위에서 마우스 버튼을 누름 |
| `mouse-up` _(macOS)_ | 아이콘 위에서 마우스 버튼을 뗌 |
| `mouse-enter` _(macOS/Windows)_ | 커서가 아이콘 안으로 들어옴 |
| `mouse-leave` _(macOS/Windows)_ | 커서가 아이콘 밖으로 나감 |
| `mouse-move` _(macOS/Windows)_ | 커서가 아이콘 위에서 움직임 |

### 풍선 알림 _(Windows)_

| 이벤트 | 발생 시점 |
|---|---|
| `balloon-show` _(Windows)_ | 풍선 알림이 표시됨 |
| `balloon-click` _(Windows)_ | 풍선 알림이 클릭됨 |
| `balloon-closed` _(Windows)_ | 풍선이 닫힘 (시간 초과 또는 사용자 해제) |
| `focus` _(Windows)_ | 포커스가 알림 영역으로 돌아옴 |

```js filename="balloon.js"
// Windows: 풍선을 띄우고 사용자가 클릭하면 반응해요
tray.displayBalloon({ title: '완료', content: '내보내기가 끝났어요.' })

tray.on('balloon-click', () => {
  mainWindow.show()
})
```

### 드래그 & 드롭 _(macOS)_

| 이벤트 | 발생 시점 |
|---|---|
| `drop` _(macOS)_ | 드래그한 아이템이 아이콘 위에 놓임 |
| `drop-files` _(macOS)_ | 파일이 아이콘 위에 놓임 (파일 경로 전달) |
| `drop-text` _(macOS)_ | 텍스트가 아이콘 위에 놓임 |
| `drag-enter` _(macOS)_ | 드래그 동작이 아이콘 안으로 들어옴 |
| `drag-leave` _(macOS)_ | 드래그 동작이 아이콘 밖으로 나감 |
| `drag-end` _(macOS)_ | 드래그 동작이 끝남 (아이콘 위 또는 다른 곳에서) |

```js filename="drop-files.js"
// macOS: 트레이 아이콘 위에 떨어뜨린 파일을 받아요
tray.on('drop-files', (event, files) => {
  // files 는 절대 경로의 배열이에요
  importFiles(files)
})
```

<EasyNote>
트레이 아이콘을 화면 구석에 놓인 작은 초인종이라고 생각해 보세요. `click` 은 누군가 그걸 누르는 것, `mouse-enter`/`mouse-leave` 는 옆을 지나가는 것이고, macOS 에서는 문 앞에 택배를 두고 갈 수도 있어요 — 그게 `drop-files` 예요.
</EasyNote>

## 인스턴스 메서드(Instance Methods)

### 핵심 (모든 플랫폼)

| 메서드 | 하는 일 |
|---|---|
| `tray.destroy()` | 트레이 아이콘을 즉시 제거 |
| `tray.setImage(image)` | 아이콘 이미지 변경 |
| `tray.setToolTip(toolTip)` | 마우스 오버 툴팁 텍스트 설정 (`''` 이면 제거) |
| `tray.setContextMenu(menu)` | 클릭/우클릭 시 보일 메뉴 설정 (`null` 이면 해제) |
| `tray.isDestroyed()` | 아이콘이 파괴되었으면 `true` |

```js filename="status-icon.js"
// 앱 상태에 맞게 아이콘을 바꿔요
tray.setImage(isOnline ? onlineIcon : offlineIcon)
tray.setToolTip(isOnline ? '연결됨' : '오프라인')
```

### 위치 · GUID · 메뉴 제어 _(macOS/Windows)_

| 메서드 | 하는 일 |
|---|---|
| `tray.getBounds()` _(macOS/Windows)_ | 아이콘의 `Rectangle` (`x`, `y`, `width`, `height`) |
| `tray.getGUID()` _(macOS/Windows)_ | 아이콘의 GUID, 설정 안 했으면 `null` |
| `tray.popUpContextMenu([menu, position])` _(macOS/Windows)_ | 컨텍스트 메뉴 띄우기 (다른 메뉴/위치 지정 가능) |
| `tray.closeContextMenu()` _(macOS/Windows)_ | 열린 컨텍스트 메뉴 닫기 |

```js filename="popup-near-icon.js"
// 트레이 아이콘 바로 아래에 커스텀 팝업 창을 띄워요
const { x, y } = tray.getBounds()
popup.setPosition(x, y)
popup.show()
```

### 타이틀 & 누름 이미지 _(macOS)_

| 메서드 | 하는 일 |
|---|---|
| `tray.setTitle(title[, options])` _(macOS)_ | 메뉴 막대 아이콘 옆에 텍스트 표시 |
| `tray.getTitle()` _(macOS)_ | 현재 타이틀 텍스트 읽기 |
| `tray.setPressedImage(image)` _(macOS)_ | 아이콘을 누르고 있는 동안 보일 이미지 |
| `tray.setIgnoreDoubleClickEvents(ignore)` _(macOS)_ | 더블클릭을 두 번의 개별 클릭으로 처리 |
| `tray.getIgnoreDoubleClickEvents()` _(macOS)_ | 더블클릭 이벤트를 무시하고 있는가? |

```js filename="title.js"
// macOS: 메뉴 막대 아이콘 옆에 실시간 카운터 표시
tray.setTitle('새 항목 3개')
```

### 풍선 알림 _(Windows)_

| 메서드 | 하는 일 |
|---|---|
| `tray.displayBalloon(options)` _(Windows)_ | 풍선 알림 표시 (`title`, `content`, `icon` ...) |
| `tray.removeBalloon()` _(Windows)_ | 현재 풍선 제거 |
| `tray.focus()` _(Windows)_ | 포커스를 알림 영역으로 되돌리기 |

<Callout type="warning">
메서드 가용성은 **플랫폼마다 달라요**. `setTitle()` 은 macOS 밖에서는 아무 일도 하지 않고, `displayBalloon()` 은 Windows 에서만 동작해요. `process.platform` 으로 분기하거나, 알림은 크로스 플랫폼인 `Notification` 모듈을 쓰는 편이 좋아요.
</Callout>

## 인스턴스 속성(Instance Properties)

`Tray` 클래스에는 **인스턴스 속성이 없어요** — 모든 상호작용은 메서드와 이벤트로 이루어져요.

---

## 쉽게 풀어 설명

`Tray` 는 앱의 **화면 구석에 있는 작은 아이콘** 이에요 — Mac 의 메뉴 막대, Windows 의 오른쪽 아래 알림 영역에 있죠. 창을 닫아도 앱에 계속 닿을 수 있게 해주는 방법이에요. 음악 플레이어, 채팅 앱, 백업 도구가 모두 여기 살아요.

사실 여기서 하는 일은 딱 세 가지예요:

1. **아이콘을 보여주기.** `new Tray(image)` 로 올리고, `setImage` 로 바꾸고(온라인/오프라인, 대기/작업 중), `setToolTip` 으로 마우스를 올렸을 때 보일 작은 라벨을 붙여요.
2. **메뉴 달기.** 대부분의 앱이 `setContextMenu(menu)` 를 써요 — 아이콘을 누르면 "열기 / 설정 / 종료" 가 나오죠. 메뉴는 `Menu` 모듈로 만들어요.
3. **클릭에 반응하기.** `click` 을 들어 창을 토글하거나, `right-click` 으로 메뉴를 띄워요. 나머지(풍선, 드래그 앤 드롭, 타이틀)는 나중에 추가할 수 있는 플랫폼별 디테일이에요.

무엇보다 한 가지 규칙을 기억하세요: **Tray 객체의 참조를 살려 두세요.** 안 그러면 사라져요. `new Tray`, `setContextMenu`, 그리고 `click` 이벤트만 익히면 현실의 거의 모든 트레이 앱을 만들 수 있어요.
