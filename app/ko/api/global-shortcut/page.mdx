`globalShortcut` 모듈은 **Main 프로세스** 에 살아요. 키보드 단축키를 운영체제에 등록해서 **앱 창이 포커스를 받고 있지 않아도** 단축키가 동작하도록 해줘요 — 다른 앱에서 작업 중일 때도 재생/일시정지 키에 반응하는 음악 플레이어처럼요.

```js filename="main.js"
const { app, globalShortcut } = require('electron')

app.whenReady().then(() => {
  // 시스템 전역 핫키 등록: Cmd/Ctrl+Shift+K
  const ok = globalShortcut.register('CommandOrControl+Shift+K', () => {
    // 다른 앱이 앞에 있어도 실행돼요
    toggleQuickCapture()
  })

  if (!ok) {
    // 이미 다른 앱이 이 조합을 쓰고 있어요
  }
})

app.on('will-quit', () => {
  // 종료 전 항상 정리해요
  globalShortcut.unregisterAll()
})
```

<Callout type="warning">
`globalShortcut` 은 앱의 `ready` 이벤트가 발생한 **이후** 에만 사용할 수 있어요 (`app.whenReady()` 사용). 그 전에 호출하면 OS 단축키 시스템이 아직 연결되지 않아 아무 의미가 없어요.
</Callout>

<EasyNote>
웹 페이지 안의 일반 키 핸들러는 그 페이지가 앞에 있을 때만 키를 들어요. **전역(global)** 단축키는 운영체제에 포스트잇을 붙이는 것과 같아요. "이 키 조합이 눌리면 — 어떤 앱이 맨 위에 있든 상관없이 — 나를 불러줘." 그래서 미디어 키가 어느 화면에서든 음악을 멈출 수 있는 거예요.
</EasyNote>

---

## 이벤트(Events)

`globalShortcut` 에는 **이벤트가 없어요**. `.on(...)` 으로 구독하지 않습니다. 대신 액셀러레이터 문자열과 콜백을 넘겨주면, 키가 눌릴 때 OS 가 그 콜백을 직접 호출해줘요.

---

## 메서드(Methods)

`globalShortcut` 객체 자체에서 호출해요 (클래스가 아니라 싱글턴이에요). **액셀러레이터(accelerator)** 는 `'CommandOrControl+Shift+K'` 같은 문자열이고, `Menu` 항목에서 쓰는 키·수식어 형식과 동일해요.

### 단축키 등록

| 메서드 | 하는 일 |
|---|---|
| `globalShortcut.register(accelerator, callback)` | 단축키 하나 등록. 성공하면 `true`, 이미 다른 앱이 쓰고 있으면 `false` 반환 (**조용히** 실패해요). |
| `globalShortcut.registerAll(accelerators, callback)` | 여러 액셀러레이터(`string[]`)를 등록하고 **하나의** 콜백을 공유. 반환값 없음. |

```js filename="register.js"
// 단축키 하나, 핸들러 하나
const ok = globalShortcut.register('CommandOrControl+X', () => {
  showOverlay()
})

if (!ok) {
  // 이미 다른 앱이 차지한 조합이에요 — 다른 키를 쓰거나 사용자에게 알려요
  notifyShortcutUnavailable('CommandOrControl+X')
}

// 여러 액셀러레이터가 하나의 핸들러를 공유 (같은 동작을 두 방식으로 트리거)
globalShortcut.registerAll(
  ['CommandOrControl+Y', 'CommandOrControl+Z'],
  () => undo()
)
```

<Callout type="warning">
등록은 **조용히 실패** 할 수 있어요. 액셀러레이터를 시스템이나 다른 앱이 이미 쓰고 있으면 `register()` 는 `false` 를 반환하고 (`registerAll()` 은 반환값이 없어요). 핫키가 살아 있다고 가정하기 전에 항상 `register()` 의 반환값을 확인하거나 `isRegistered()` 로 검증하세요.
</Callout>

### 확인 & 해제

| 메서드 | 하는 일 |
|---|---|
| `globalShortcut.isRegistered(accelerator)` | **내 앱** 이 현재 이 단축키를 쥐고 있으면 `true`. 다른 앱이 갖고 있으면, 방금 등록을 시도했더라도 `false`. |
| `globalShortcut.unregister(accelerator)` | 단축키 하나를 해제해서 키를 OS / 다른 앱에 돌려줘요. |
| `globalShortcut.unregisterAll()` | 내 앱이 등록한 모든 단축키를 해제. |

```js filename="unregister.js"
// 우리 핫키가 실제로 활성 상태인가?
if (globalShortcut.isRegistered('CommandOrControl+Shift+K')) {
  // 우리 것이에요 — 믿고 사용해도 됨
}

// 단축키 하나 돌려주기
globalShortcut.unregister('CommandOrControl+Shift+K')

// 전부 정리 (will-quit 에서 하세요)
globalShortcut.unregisterAll()
```

<Callout type="warning">
`app.on('will-quit', ...)` 핸들러에서 항상 `globalShortcut.unregisterAll()` 을 호출하세요. 단축키는 자동으로 해제되지 않으며, 등록된 채로 두면 다른 앱의 키 조합을 계속 가로챌 수 있어요.
</Callout>

### 일시 중단

| 메서드 | 하는 일 |
|---|---|
| `globalShortcut.setSuspended(suspended)` | 등록을 해제하지 않고 **모든** 단축키를 한 번에 중단(`true`)/재개(`false`). 중단 중에는 새 등록도 실패해요. |
| `globalShortcut.isSuspended()` | 단축키 처리가 현재 중단되어 있으면 `true`. |

```js filename="suspend.js"
// "핫키 다시 지정" 설정 화면: 전역 단축키를 잠시 멈춰서
// 사용자가 누르는 키가 핸들러가 아닌 설정 UI 로 가도록 해요.
globalShortcut.setSuspended(true)

// ...사용자가 새 조합을 입력하게 둬요...

globalShortcut.setSuspended(false) // 재개 — 단축키들이 다시 살아나요
```

<EasyNote>
`setSuspended(true)` 는 아무것도 뽑지 않고 마스터 스위치만 "끄기" 로 내리는 것과 같아요. 단축키는 여전히 등록되어 기억되고 있어요 — 다시 켤 때까지 발동만 멈출 뿐이죠. 사용자가 새 핫키를 고르게 하는 설정 화면에 딱이에요.
</EasyNote>

<Callout type="info">
**macOS 미디어 키 (10.14+):** `MediaPlayPause`, `MediaNextTrack`, `MediaPreviousTrack`, `MediaStop` 같은 미디어 액셀러레이터를 등록하려면, 앱이 **신뢰된 접근성 클라이언트(trusted accessibility client)** 여야 해요. 그렇지 않으면 등록이 동작하지 않습니다. _(Windows)_ 에서는 이런 접근성 요건이 없어서 같은 미디어 액셀러레이터가 그냥 등록돼요 — 다만 `Win+L`(화면 잠금) 처럼 OS 가 차지한 시스템 예약 조합은 가져올 수 없어요.
</Callout>

---

## 속성(Properties)

`globalShortcut` 에는 **속성이 없어요** — 오직 메서드의 집합이에요. 상태(어떤 단축키가 활성인지, 처리가 중단됐는지)는 속성이 아니라 `isRegistered()` 와 `isSuspended()` 로 읽어요.

---

## 쉽게 풀어 설명

`globalShortcut` 은 한 가지 질문에 답해요. **"내 앱이 백그라운드에 있어도, 사용자가 키 조합을 누르면 코드를 실행하려면 어떻게 하지?"**

모듈 전체가 머릿속에 담기는 작은 흐름이에요:

1. **ready 를 기다려요.** 모든 건 `app.whenReady()` 안에서 시작돼요 — OS 단축키 시스템은 그 전에는 쓸 수 없어요.
2. **등록해요.** `register('CommandOrControl+Shift+K', callback)` 으로 핫키를 운영체제에 넘겨요. 차지했으면 `true`, 이미 누가 갖고 있으면 `false` 를 돌려줘요. 한 동작에 여러 키가 필요하면 `registerAll([...], callback)`.
3. **필요하면 확인해요.** `isRegistered(...)` 로 그 조합이 정말 내 것인지 확인하고, `isSuspended()` 로 처리가 멈춰 있는지 알 수 있어요.
4. **잃지 않고 멈춰요.** `setSuspended(true)` 는 모든 단축키를 조용히 음소거하고 ("새 핫키를 눌러보세요" 설정 화면에 유용해요), `setSuspended(false)` 로 되살려요.
5. **정리해요.** 하나는 `unregister(...)`, 전부는 `unregisterAll()` — 그리고 OS 가 대신 치워주지 않으니 후자는 항상 `will-quit` 에서 해주세요.

**키보드를 위한 안내 데스크 직원** 이라고 생각해보세요. 메모를 남기면("누가 Cmd+Shift+K 를 누르면 나를 불러줘"), 직원은 당신이 자리에 있든 없든 문을 지켜봐요. 그리고 떠나기 전에 메모를 찢어 다음 사람이 깨끗한 책상을 받도록 하는 거죠.
