`pushNotifications` 모듈은 **Main 프로세스** 에 살아요. 앱을 Apple 의 **APNS** 서버에 등록하고, 네트워크를 통해 도착하는 알림을 듣게 해줘요 — 창이 백그라운드에 있어도 뜨는 그런 알림이에요.

```js filename="main.js"
const { pushNotifications } = require('electron')

app.whenReady().then(async () => {
  // Apple 에 기기 토큰을 요청해요 — 서버는 이 토큰으로 이 기기를 지정해요
  const token = await pushNotifications.registerForAPNSNotifications()
  sendTokenToYourServer(token)
})
```

<Callout type="info">
`pushNotifications` 는 **macOS 전용** 이에요. Windows·Linux 에서는 모듈이 존재하긴 하지만 메서드와 이벤트가 아무 동작도 하지 않아요. 항상 `process.platform === 'darwin'` 으로 감싸 주세요.
</Callout>

<EasyNote>
APNS 를 **앱 전용 우체국** 이라고 생각해 보세요. 먼저 우체국에 주소를 알려줘요(`registerForAPNSNotifications` → 토큰을 받음). 그 주소를 백엔드에 넘기면, 그때부터 백엔드는 바로 이 기기로만 편지(알림)를 부칠 수 있어요 — 앱이 조용히 백그라운드에 있어도요.
</EasyNote>

## 이벤트(Events)

`pushNotifications.on('이벤트명', 핸들러)` 로 듣습니다. 여기 있는 이벤트는 모두 _(macOS)_ 전용이에요.

| 이벤트 | 발생 시점 |
|---|---|
| `received-apns-notification` _(macOS)_ | 앱이 실행 중일 때 원격 APNS 알림이 도착함. 알림 페이로드가 담긴 `userInfo` 객체(`Record<string, unknown>`)를 전달해 줌 |

```js filename="receive.js"
// 백엔드가 APNS 를 통해 보낸 알림에 반응해요
pushNotifications.on('received-apns-notification', (event, userInfo) => {
  // userInfo 는 서버가 보낸 페이로드예요 (aps + 직접 넣은 커스텀 키)
  handleIncomingNotification(userInfo)
})
```

<EasyNote>
`received-apns-notification` 는 우편함 벨이에요. 앱이 실제로 실행 중일 때만 울려요 — 건네주는 `userInfo` 는 서버가 봉투 안에 넣어 둔 내용 그대로예요.
</EasyNote>

## 메서드(Methods)

### 등록 _(macOS)_

| 메서드 | 하는 일 |
|---|---|
| `pushNotifications.registerForAPNSNotifications()` _(macOS)_ | 앱을 APNS 에 등록해요. **기기 토큰** 으로 resolve 되는 `Promise<string>` 을 반환해요(실패 시 에러 문자열로 reject). 이 토큰을 서버에서 써서 이 기기를 지정해요 |
| `pushNotifications.unregisterForAPNSNotifications()` _(macOS)_ | 앱을 APNS 에서 등록 해제해요. `void` 를 반환해요. 나중에 `registerForAPNSNotifications()` 를 다시 호출해 재등록할 수 있어요 |

```js filename="register.js"
async function setupPush() {
  // macOS 가 아니면 전부 건너뛰어요
  if (process.platform !== 'darwin') return

  try {
    const token = await pushNotifications.registerForAPNSNotifications()
    // 토큰은 백엔드가 "이 기기"를 지정하는 수단이에요 — 서버에 저장하세요
    await api.saveDeviceToken(token)
  } catch (err) {
    // 등록은 실패할 수 있어요 (엔타이틀먼트 없음, 오프라인, 샌드박스 문제 ...)
    reportPushError(err)
  }
}

function teardownPush() {
  if (process.platform !== 'darwin') return
  // 알림 수신을 중단해요 (예: 사용자가 로그아웃함)
  pushNotifications.unregisterForAPNSNotifications()
}
```

<Callout type="warning">
APNS 는 **올바르게 코드 서명되고 엔타이틀먼트가 부여된** 빌드에서만 동작해요. 서명되지 않은 개발 빌드, 푸시 알림 엔타이틀먼트가 없는 앱, APNS 프로비저닝이 안 된 앱에는 토큰이 발급되지 않아요. 서명된 빌드에서 푸시를 테스트하고, reject 된 `Promise` 를 항상 처리하세요.
</Callout>

## 쉽게 풀어 설명

`pushNotifications` 는 **macOS 전용으로 Apple 푸시 서비스에 들어가는 문** 이에요. 흐름은 짧고, 보통 한 번만 설정하면 돼요.

1. **내 주소를 받아요.** `registerForAPNSNotifications()` 를 호출하면 Apple 이 **기기 토큰** 을 돌려줘요 — "바로 이 Mac, 바로 이 앱" 을 뜻하는 고유 문자열이에요. 이 토큰을 백엔드로 보내세요.
2. **편지를 기다려요.** 백엔드가 그 토큰으로 알림을 보내달라고 Apple 에 요청해요. 앱이 실행 중이면 `received-apns-notification` 이벤트가 발생하고 페이로드(`userInfo`)를 건네줘요.
3. **다 쓰면 끊어요.** 사용자가 로그아웃하거나 더 이상 푸시가 필요 없으면 `unregisterForAPNSNotifications()` 를 호출하세요.

두 가지만 꼭 기억하세요. **macOS 전용** 이라는 점(`process.platform === 'darwin'` 으로 감싸기), 그리고 토큰을 실제로 받으려면 **서명되고 엔타이틀먼트가 부여된 빌드** 가 필요하다는 점이에요 — 그래서 등록 호출은 항상 `try/catch` 로 감싸세요.

이름 두 개만 외우면 끝이에요: **`registerForAPNSNotifications()`**(주소 받기) 와 **`received-apns-notification`**(초인종). 나머지는 여러분의 서버 로직이에요.
