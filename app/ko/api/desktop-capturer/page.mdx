`desktopCapturer` 모듈은 **Main 프로세스** 에 살아요. 답하는 질문은 딱 하나예요 — _"지금 캡처할 수 있는 화면과 창은 무엇인가?"_ 그리고 각 소스의 `id` 를 담은 목록을 돌려줘요. 그 `id` 를 렌더러의 `getUserMedia` / `getDisplayMedia` 에 넘기면 실제 영상을 가져올 수 있어요.

```js filename="main.js"
const { desktopCapturer } = require('electron')

const sources = await desktopCapturer.getSources({ types: ['screen', 'window'] })
// 각 소스: { id, name, thumbnail, display_id, appIcon }
sources.forEach((source) => {
  // source.id -> 렌더러에서 chromeMediaSourceId 로 사용
})
```

<EasyNote>
`desktopCapturer` 는 **채널 안내표** 이지 TV 가 아니에요. 어떤 화면과 창이 존재하는지 알려 주고, 각각에 이름·썸네일·`id` 를 붙여 줄 뿐이에요. 소스를 실제 영상 스트림으로 바꾸는 일은 렌더러에서 `getUserMedia` / `getDisplayMedia` 가 해요 — `desktopCapturer` 는 "무엇을 캡처할지"만 가리켜 줘요.
</EasyNote>

## 메서드(Methods)

이 모듈에는 메서드가 **단 하나** 있어요. 이벤트도 속성도 없습니다.

| 메서드 | 하는 일 |
|---|---|
| `desktopCapturer.getSources(options)` | 캡처 가능한 화면/창 전체를 담은 `Promise<DesktopCapturerSource[]>` 반환 |

### `getSources` 옵션

| 옵션 | 타입 | 하는 일 |
|---|---|---|
| `types` | `string[]` | **필수.** 가져올 종류: `'screen'`, `'window'`, 또는 둘 다 |
| `thumbnailSize` | `Size` | 원하는 썸네일 크기. 기본값 `150×150`. width/height 를 `0` 으로 두면 썸네일 생성을 건너뛰어 더 빨라요 |
| `fetchWindowIcons` | `boolean` | `true` 면 각 창의 `appIcon` 을 채워요. 기본값 `false`. 화면 소스에서는 항상 `null` |

```js filename="get-sources.js"
const { desktopCapturer } = require('electron')

// 모든 화면·창 나열, 속도를 위해 썸네일 생략, 앱 아이콘 포함
const sources = await desktopCapturer.getSources({
  types: ['screen', 'window'],
  thumbnailSize: { width: 0, height: 0 },
  fetchWindowIcons: true,
})

const primaryScreen = sources.find((s) => s.name === 'Entire Screen')
```

### `DesktopCapturerSource` 객체

반환된 배열의 각 항목은 캡처 가능한 소스 하나를 설명해요.

| 필드 | 타입 | 무엇인가 |
|---|---|---|
| `id` | `string` | 소스 식별자 — 렌더러에서 `chromeMediaSourceId` 제약으로 사용. 형식: `window:XX:YY` 또는 `screen:ZZ:0` |
| `name` | `string` | 화면은 `Entire Screen` 또는 `Screen <번호>`, 창은 그 창의 제목과 일치 |
| `thumbnail` | `NativeImage` | 미리보기 이미지. 요청한 `thumbnailSize` 와 크기가 일치한다는 **보장은 없음** |
| `display_id` | `string` | `screen` API 의 해당 Display `id` 와 일치. 알 수 없으면 빈 문자열 |
| `appIcon` | `NativeImage` | 창을 소유한 앱의 아이콘 — 화면 소스에서는 `null` (`fetchWindowIcons: true` 일 때만 채워짐) |

### 권장 흐름: `setDisplayMediaRequestHandler`

Electron 17 부터 화면 공유를 켜는 가장 안전한 방법은, **렌더러** 가 표준 웹 API `navigator.mediaDevices.getDisplayMedia()` 를 호출하고, **메인** 프로세스가 `desktopCapturer` 의 소스로 그 요청에 답하는 거예요.

```js filename="picker.js"
const { session, desktopCapturer } = require('electron')

session.defaultSession.setDisplayMediaRequestHandler(
  (request, callback) => {
    desktopCapturer.getSources({ types: ['screen'] }).then((sources) => {
      // 첫 번째 화면을 허용, 시스템(루프백) 오디오와 함께
      callback({ video: sources[0], audio: 'loopback' })
    })
  },
  // 가능한 플랫폼에서는 OS 자체 네이티브 선택기를 띄움 (macOS/Windows)
  { useSystemPicker: true }
)
```

```js filename="renderer.js"
// 렌더러 프로세스 — 표준 웹 API 가 위 핸들러를 호출시켜요
const stream = await navigator.mediaDevices.getDisplayMedia({
  audio: true,
  video: { width: 320, height: 240, frameRate: 30 },
})
```

<EasyNote>
`setDisplayMediaRequestHandler` 는 **문 앞의 경비원** 이라고 생각하세요. 웹 페이지가 표준 `getDisplayMedia()` 로 "화면 녹화해도 될까요?" 하고 정중히 물으면, 메인 프로세스가 어떤 소스를 내줄지 결정해요 — 페이지가 화면에 직접 손대게 두지 않고요.
</EasyNote>

<Callout type="warning">
`getSources()` 는 사용자가 사적이라 여길 수 있는 창과 화면을 나열해요. 권한이 필요한 기능처럼 다루세요 — 절대 자동으로 캡처를 허용하지 말고, 선택기를 띄워 사용자가 고르게 하고, 창 제목(`source.name`)은 로그에 남기지 마세요. 민감한 내용이 새어 나갈 수 있어요.
</Callout>

<Callout type="warning">
**macOS** 는 사용자가 시스템 설정에서 **화면 기록** 권한을 허용해야 해요. 먼저 `systemPreferences.getMediaAccessStatus('screen')` 으로 확인하세요. `'granted'` 가 아니면 캡처가 검은 화면을 반환해요. macOS 14.2+ 에서 **시스템 오디오** 를 캡처하려면 `Info.plist` 에 `NSAudioCaptureUsageDescription` 가 필요해요.
</Callout>

<Callout type="info">
**Windows** 에는 요청해야 할 OS 수준의 화면 기록 권한이 따로 없어요 — `getSources()` 는 사전 허용 없이도 동작해요. 다만 네이티브 시스템 선택기(`useSystemPicker: true`)는 **Windows 11**(22H2 이상)이 필요해요. Windows 10 에서는 무시되니, `getSources()` 목록으로 직접 선택기 UI 를 만들어 대비하세요. 루프백 **시스템 오디오**(`audio: 'loopback'`)는 Windows 에서 지원되며, `Info.plist` 같은 별도 선언은 필요 없어요.
</Callout>

<Callout type="warning">
**Linux (PipeWire/Wayland)** 는 OS 포털을 통해 선택된 **단 하나** 의 소스를 반환해요 — macOS·Windows 처럼 모든 창/화면을 나열할 수 없어요. 전체 목록이 아니라 하나의 소스를 받는다고 가정하고 UI 를 설계하세요.
</Callout>

## 이벤트(Events)

`desktopCapturer` 는 이벤트를 **발생시키지 않아요.** 한 번 호출하고 결과를 기다리는 단발성 질의 모듈이에요 — `getSources()` 를 호출하고 await 하면 끝이에요.

## 속성(Properties)

`desktopCapturer` 는 노출하는 **속성이 없어요.**

## 쉽게 풀어 설명

`desktopCapturer` 는 작지만 중요한 일 하나만 해요 — **녹화해도 되는 것들의 메뉴판** 을 건네주는 거예요. `getSources({ types: ['screen', 'window'] })` 를 호출하면 목록이 돌아와요. 모든 모니터, 열려 있는 모든 창이, 각각 친절한 `name`, 작은 `thumbnail` 미리보기, 고유한 `id` 와 함께요.

핵심은 바로 그 `id` 예요. `desktopCapturer` 자체는 아무것도 녹화하지 않아요. `id` 를 렌더러에 넘기면, 거기서 브라우저의 `getUserMedia`/`getDisplayMedia` 가 그것을 진짜 살아 있는 영상 스트림으로 바꿔요. 메인 프로세스는 _무엇_ 을 고르고, 렌더러가 _그것_ 을 캡처해요.

새로 만드는 기능이라면 최신 방식을 권해요 — 페이지가 표준 `navigator.mediaDevices.getDisplayMedia()` 를 호출하게 하고, 메인 프로세스에서 `session.setDisplayMediaRequestHandler` 로 답하세요(필요하면 `useSystemPicker: true` 로 OS 자체 선택기를 띄우고요). 그리고 사람을 잊지 마세요 — macOS 에서는 사용자가 먼저 화면 기록 권한을 허용해야 하고, 모든 플랫폼에서 무엇을 공유할지는 사용자가 고르게 해야 해요.

이름 두 개면 거의 다예요: **`getSources()`**(무엇을 캡처할 수 있나?) 와 **`setDisplayMediaRequestHandler`**(페이지에 안전하게 건네주려면?).
