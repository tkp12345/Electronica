`powerMonitor` 모듈은 **Main 프로세스** 에 살아요. OS 의 전원 안내 데스크예요. 컴퓨터가 잠들거나 깨어날 때, 배터리와 콘센트 전원을 오갈 때, 화면을 잠그거나 풀 때, 또는 과열되기 시작할 때를 알려줘요.

```js filename="main.js"
const { app, powerMonitor } = require('electron')

// powerMonitor 는 'ready' 이벤트 이후에만 사용할 수 있어요
app.whenReady().then(() => {
  powerMonitor.on('suspend', () => {
    // 컴퓨터가 잠들려는 중 — 작업 멈추고 연결 정리
    pauseSyncing()
  })
})
```

<Callout type="warning">
`powerMonitor` 는 `app` 의 `ready` 이벤트 **이후** 에만 require·사용할 수 있어요. 그 전에(메인 스크립트 최상단, `whenReady()` 이전) 건드리면 에러가 나요.
</Callout>

<EasyNote>
`powerMonitor` 는 앱이 돌아가는 컴퓨터의 **계기판 경고등 묶음** 이에요. 스스로 무언가를 바꾸지는 않고, "지금 잠들고 있어요", "이제 배터리로 돌아가요", "과열되고 있어요" 라고 불을 켜서 알려줄 뿐이에요. 그러면 코드가 알맞게 반응하면 돼요.
</EasyNote>

## 이벤트(Events)

`powerMonitor.on('이벤트명', 핸들러)` 로 듣습니다. _(macOS)_ / _(Windows)_ / _(Linux)_ 표시는 해당 플랫폼에서만 발생해요.

### 절전 & 깨어남

| 이벤트 | 발생 시점 |
|---|---|
| `suspend` | 시스템이 절전(잠자기)에 들어가는 중 |
| `resume` | 시스템이 절전에서 깨어나는 중 |
| `shutdown` _(Linux/macOS)_ | 시스템이 재부팅·종료되려는 시점 — `event.preventDefault()` 지원 |

```js filename="sleep.js"
powerMonitor.on('suspend', () => {
  // 곧 잠들어요 — 상태 저장, 소켓 정리
  saveDraft()
})

powerMonitor.on('resume', () => {
  // 다시 깨어났어요 — 재연결 및 재동기화
  reconnect()
})

powerMonitor.on('shutdown', (event) => {
  // Linux/macOS: 정리할 시간만큼 종료를 미뤄요
  event.preventDefault()
  finishCriticalWork().then(() => app.quit())
})
```

### 전원 종류

| 이벤트 | 발생 시점 |
|---|---|
| `on-ac` _(macOS/Windows)_ | AC(콘센트) 전원으로 전환됨 |
| `on-battery` _(macOS/Windows)_ | 배터리 전원으로 전환됨 |

```js filename="power-source.js"
powerMonitor.on('on-battery', () => {
  // 배터리로 동작 중 — 에너지 절약을 위해 백그라운드 작업 줄이기
  setSyncInterval('low')
})

powerMonitor.on('on-ac', () => {
  // 다시 콘센트에 연결됨 — 정상 속도로 복귀
  setSyncInterval('normal')
})
```

<EasyNote>
`on-battery` / `on-ac` 가 바로 좋은 앱이 배터리를 아끼는 방법이에요. 사용자가 플러그를 뽑는 순간 애니메이션을 줄이고, 폴링을 늦추고, 무거운 백그라운드 작업을 멈춰요.
</EasyNote>

### 화면 잠금 & 세션

| 이벤트 | 발생 시점 |
|---|---|
| `lock-screen` _(macOS/Windows)_ | 시스템이 화면을 잠그려는 시점 |
| `unlock-screen` _(macOS/Windows)_ | 화면 잠금이 막 풀린 시점 |
| `user-did-become-active` _(macOS)_ | 사용자 로그인 세션이 활성화됨 |
| `user-did-resign-active` _(macOS)_ | 사용자 로그인 세션이 비활성화됨 |

```js filename="lock.js"
powerMonitor.on('lock-screen', () => {
  // 사용자가 자리를 비움 — 민감한 UI 가리고, 복귀 시 재인증 요구
  blurSensitiveData()
})

powerMonitor.on('unlock-screen', () => {
  promptForReauth()
})
```

### 발열 & CPU (macOS / Windows)

| 이벤트 | 발생 시점 |
|---|---|
| `thermal-state-change` _(macOS)_ | 발열 상태가 바뀜 — 핸들러가 `details.state` 를 받음: `'unknown'` \| `'nominal'` \| `'fair'` \| `'serious'` \| `'critical'` |
| `speed-limit-change` _(macOS/Windows)_ | CPU 속도 제한이 바뀜 — 핸들러가 `details.limit`(숫자, `0`–`100`, 최대 속도 대비 퍼센트)을 받음 |

```js filename="thermal.js"
powerMonitor.on('thermal-state-change', (details) => {
  // macOS: Mac 이 뜨거워지는 중 — 무거운 연산을 줄여요
  if (details.state === 'serious' || details.state === 'critical') {
    pauseHeavyTasks()
  }
})

powerMonitor.on('speed-limit-change', (details) => {
  // details.limit 은 현재 허용된 CPU 최대 속도의 퍼센트예요
  adjustWorkload(details.limit)
})
```

## 메서드(Methods)

### 유휴 상태 감지

| 메서드 | 플랫폼 | 하는 일 |
|---|---|---|
| `powerMonitor.getSystemIdleState(idleThreshold)` | 전체 | `'active'`, `'idle'`, `'locked'`, `'unknown'` 중 하나를 반환. `idleThreshold` 는 유휴로 간주할 무입력 **초(seconds)** 수 |
| `powerMonitor.getSystemIdleTime()` | 전체 | 시스템 유휴 시간을 **초(seconds)** 단위로 반환 (마지막 사용자 입력 이후 경과 시간) |

```js filename="idle.js"
// 입력이 60초 동안 없으면 사용자를 "유휴" 로 간주
const state = powerMonitor.getSystemIdleState(60)
// 'active' | 'idle' | 'locked' | 'unknown'

if (state === 'idle') {
  pauseNotifications()
}

const seconds = powerMonitor.getSystemIdleTime() // 예: 12
```

<Callout type="warning">
`getSystemIdleState` 와 `getSystemIdleTime` 은 밀리초가 아니라 **초(seconds)** 를 받고 반환해요. 밀리초 값(예: `60000`)을 넘기면 사용자가 절대 유휴로 판단되지 않아요.
</Callout>

### 전원 & 발열 상태

| 메서드 | 플랫폼 | 하는 일 |
|---|---|---|
| `powerMonitor.isOnBatteryPower()` | 전체 | 시스템이 현재 배터리로 동작 중이면 `true` 반환 |
| `powerMonitor.getCurrentThermalState()` | macOS | 현재 발열 상태 반환: `'unknown'` \| `'nominal'` \| `'fair'` \| `'serious'` \| `'critical'` |

```js filename="battery.js"
// 창이 열리는 순간 얼마나 공격적으로 동작할지 결정
if (powerMonitor.isOnBatteryPower()) {
  enablePowerSaverMode()
}

// macOS 전용: 이미 뜨거우면 비싼 렌더링을 건너뛰기
if (powerMonitor.getCurrentThermalState() === 'critical') {
  skipBackgroundRender()
}
```

## 속성(Properties)

| 속성 | 타입 | 무엇인가 |
|---|---|---|
| `powerMonitor.onBatteryPower` | `boolean` (읽기전용) | 시스템이 배터리로 동작 중이면 `true` — `isOnBatteryPower()` 의 속성 버전으로, 호출 없이 한 번 읽을 때 편리 |

```js filename="property.js"
// 메서드 호출 없이 빠르게 한 번 읽기
if (powerMonitor.onBatteryPower) {
  dimUI()
}
```

## 쉽게 풀어 설명

`powerMonitor` 는 **"지금 이 컴퓨터의 컨디션이 어떤가요?"** 를 알려주는 모듈이에요. 컴퓨터를 직접 바꾸지는 않고, 상태를 보고해 줄 뿐이라 앱이 그에 맞게 예의 바르게 행동할 수 있게 해줘요.

크게 세 가지 질문에 답해요:

1. **컴퓨터가 깨어 있나요?** `suspend`(잠드는 중)와 `resume`(깨어나는 중)을 들으세요. `suspend` 때 작업을 저장하고 네트워크 연결을 정리하고, `resume` 때 재연결·재동기화하면 돼요. Linux/macOS 에서는 `shutdown` 을 잠시 미룰 수도 있어요.
2. **사용자가 자리에 있고, 어떤 전원을 쓰나요?** `getSystemIdleTime()` / `getSystemIdleState(초)` 는 자리를 비웠는지 알려주고, `lock-screen` / `unlock-screen` 은 화면을 잠갔는지 알려주며, `isOnBatteryPower()` 와 `on-battery` / `on-ac` 이벤트는 플러그를 뽑았을 때 작업을 줄이라고 알려줘요.
3. **과열되고 있나요?** macOS 에서는 `thermal-state-change` 와 `getCurrentThermalState()` 로, OS 가 스로틀링하기 전에 무거운 연산을 미리 줄일 수 있어요.

황금률은 이거예요. **`app` 의 `ready` 를 기다린 뒤에 건드릴 것**, 그리고 유휴 관련 값은 전부 **초(seconds)** 단위라는 것. `suspend`/`resume`, `on-battery`/`on-ac`, `getSystemIdleState` 만 익히면 실무에서 만나는 거의 모든 경우를 다룰 수 있어요.
