`session` 모듈은 **Main 프로세스** 에 살아요. 앱이 불러오는 웹 페이지의 _저장소와 네트워크_ 를 제어해요 — 쿠키, 캐시, 프록시, 권한, 다운로드 등등이요. 모든 `BrowserWindow` 와 `webContents` 는 어떤 `Session` 안에서 돌아가고, 가장 흔히 쓰는 세션은 `session.defaultSession` 으로 접근해요.

```js filename="main.js"
const { session, app } = require('electron')

app.whenReady().then(() => {
  const ses = session.defaultSession

  // 기본 세션의 HTTP 캐시를 모두 비워요
  ses.clearCache()
})
```

## 세션을 얻는 방법

`new Session()` 을 직접 쓰는 일은 없어요. `Session` 은 항상 건네받아요 — 공유되는 기본 세션이거나, 직접 요청한 파티션/경로 기반 세션이죠. 각 `webContents` 도 자기만의 `webContents.session` 을 가지고 있어요.

| `Session` 얻는 방법 | 무엇을 얻나 |
|---|---|
| `session.defaultSession` | 공유되는 기본 세션 (`app.whenReady()` 이후) |
| `session.fromPartition(partition[, options])` | 이름 있는 세션 — `'persist:이름'` 은 재시작 후에도 유지, `'이름'` 은 메모리에만 |
| `session.fromPath(path[, options])` | 절대 경로에 저장되는 세션 |
| `webContents.session` | 특정 페이지/창이 속한 세션 |

```js filename="partitions.js"
// 격리된 메모리 전용 세션 ("비공개" 창에 좋아요)
const incognito = session.fromPartition('temp-window')

// 재시작 후에도 쿠키·캐시가 유지되는 지속 세션
const account = session.fromPartition('persist:account-1')
```

<EasyNote>
`session` 은 브라우저 탭 뒤에 있는 **프로필** 이라고 생각하세요. 프로필을 공유하는 두 탭은 로그인·기록을 공유하고, 프로필을 바꾸면 깨끗한 상태로 시작해요. `defaultSession` 은 매일 쓰는 프로필이고, `fromPartition('temp')` 는 시크릿 창이에요.
</EasyNote>

## 인스턴스 이벤트(Instance Events)

`ses.on('이벤트명', 핸들러)` 로 듣습니다. _(macOS)_ / _(Windows)_ / _(Linux)_ 표시는 해당 플랫폼에서만 발생해요.

### 다운로드

| 이벤트 | 발생 시점 |
|---|---|
| `will-download` | `webContents` 의 리소스를 다운로드하기 직전 — `DownloadItem` 을 전달 |

```js filename="download.js"
session.defaultSession.on('will-download', (event, item, webContents) => {
  // 저장 대화상자 없이 모든 다운로드를 고정 폴더로
  item.setSavePath('/tmp/downloads/' + item.getFilename())

  item.on('done', (e, state) => {
    if (state === 'completed') notifyDownloadDone(item.getSavePath())
  })
})
```

### 확장 프로그램

| 이벤트 | 발생 시점 |
|---|---|
| `extension-loaded` | Chrome 확장 프로그램 로딩 완료 (`Extension` 전달) |
| `extension-ready` | 확장 프로그램 백그라운드 페이지가 초기화되어 준비됨 |
| `extension-unloaded` | 확장 프로그램이 언로드됨 |

### 디바이스 (HID / Serial / USB)

| 이벤트 | 발생 시점 |
|---|---|
| `select-hid-device` | 페이지가 HID 디바이스를 요청해 하나를 골라야 함 |
| `hid-device-added` | `select-hid-device` 프롬프트 중 HID 디바이스가 추가됨 |
| `hid-device-removed` | `select-hid-device` 프롬프트 중 HID 디바이스가 제거됨 |
| `hid-device-revoked` | 허용했던 HID 디바이스 접근이 취소됨 |
| `select-serial-port` | 페이지가 시리얼 포트를 요청해 하나를 골라야 함 |
| `serial-port-added` | `select-serial-port` 프롬프트 중 시리얼 포트가 추가됨 |
| `serial-port-removed` | `select-serial-port` 프롬프트 중 시리얼 포트가 제거됨 |
| `serial-port-revoked` | 허용했던 시리얼 포트 접근이 취소됨 |
| `select-usb-device` | 페이지가 USB 디바이스를 요청해 하나를 골라야 함 |
| `usb-device-added` | `select-usb-device` 프롬프트 중 USB 디바이스가 추가됨 |
| `usb-device-removed` | `select-usb-device` 프롬프트 중 USB 디바이스가 제거됨 |
| `usb-device-revoked` | 허용했던 USB 디바이스 접근이 취소됨 |
| `select-webauthn-account` | WebAuthn 계정을 선택해야 함 |

```js filename="select-device.js"
// 선택 UI 를 띄우지 않고 시리얼 포트를 자동 선택
session.defaultSession.on('select-serial-port', (event, portList, webContents, callback) => {
  event.preventDefault()
  const usbStick = portList.find((p) => p.vendorId === '0x2341')
  callback(usbStick ? usbStick.portId : '') // '' = 요청 취소
})
```

## 쉽게 풀어 설명

`session` 은 앱이 불러오는 모든 페이지 뒤에 있는 **프로필** 이에요. 브라우저 프로필이 가진 것들을 그대로 소유하죠 — 쿠키(로그인), 캐시(속도), 권한(카메라/마이크), 다운로드, 네트워크 규칙(프록시·user-agent) 까지요.

크게 세 가지 개념이면 거의 다 돼요:

1. **어떤 프로필?** 대부분 앱은 그냥 `session.defaultSession` 을 써요. 격리된 일회용 프로필이 필요하면 `session.fromPartition('temp')`, 재시작 후에도 유지하려면 `session.fromPartition('persist:이름')` 이에요.
2. **하위 객체들이 무거운 일을 해요.** 쿠키는 `ses.cookies`, 요청 가로채기는 `ses.webRequest`, 커스텀 URL 스킴은 `ses.protocol` 에 있어요. 세션은 허브이고, 이들은 바퀴살이에요.
3. **보안 핸들러가 여기서 돌아가요.** `setPermissionRequestHandler`, `setCertificateVerifyProc`, `select-usb-device` 계열 — 웹 콘텐츠가 무엇을 만질 수 있고 없는지를 여기서 결정해요.

세 이름만 기억하면 일상은 끝나요: **`defaultSession`**(보통 원하는 그 프로필), **`ses.cookies`**(로그인 읽기/쓰기), **`setPermissionRequestHandler`**(페이지가 할 수 있는 일 잠그기). 나머지는 찾아보면 돼요 — 이 페이지가 바로 그 용도예요.
