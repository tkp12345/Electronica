`netLog` 모듈은 **Main 프로세스** 에 살아요. `session` 의 원시 네트워크 활동 — DNS 조회, 소켓 연결, 요청, 리다이렉트, 프록시 해석 — 을 JSON 로그 파일로 기록하고, 이 파일을 [netlog-viewer](https://netlog-viewer.appspot.com/) 로 열어 까다로운 연결 문제를 디버깅할 수 있어요.

```js filename="main.js"
const { app, netLog } = require('electron')

app.whenReady().then(async () => {
  await netLog.startLogging('/path/to/net-log')
  // ... 여기서 네트워크 활동이 일어나요 ...
  const path = await netLog.stopLogging()
  // 'Net-logs written to', path
})
```

<EasyNote>
`netLog` 은 앱 네트워크의 **블랙박스(비행 기록 장치)** 예요. 녹화를 누르고, 앱이 요청을 보내게 둔 뒤, 멈추면 — 무언가 잘못됐을 때 회선 위에서 정확히 무슨 일이 있었는지 순서대로 보여 주는 기록 파일이 남아요.
</EasyNote>

---

## 이벤트(Events)

`netLog` 모듈은 **이벤트를 발생시키지 않아요**. 두 개의 메서드로 제어하고 한 개의 속성을 읽는 게 전부라, 구독할 이벤트가 없어요.

---

## 메서드(Methods)

모든 `netLog` 메서드는 `app` 의 `ready` 이벤트가 발생한 **이후** 에 호출해야 해요.

### 로깅 제어

| 메서드 | 하는 일 |
|---|---|
| `netLog.startLogging(path[, options])` | `path` 의 파일에 네트워크 이벤트 기록 시작 — 기록이 시작되면 resolve 되는 `Promise<void>` 반환 |
| `netLog.stopLogging()` | 기록을 멈추고 로그를 디스크에 flush — 기록이 끝나면 값 없이 resolve 되는 `Promise<void>` 반환 |

`startLogging` 은 선택적 `options` 객체를 받아요:

| 옵션 | 타입 | 하는 일 |
|---|---|---|
| `captureMode` | `string` | 얼마나 많은 데이터를 기록할지: `'default'`, `'includeSensitive'`(쿠키·인증 헤더 포함), `'everything'`(요청/응답 본문 전체) |
| `maxFileSize` | `number` | 파일이 이 바이트 수를 넘으면 자동으로 로깅 중지 |

```js filename="capture.js"
const { netLog } = require('electron')

// 요청/응답 본문까지 전부 기록, 100 MB 로 상한
await netLog.startLogging('/tmp/net-log.json', {
  captureMode: 'everything',
  maxFileSize: 100 * 1024 * 1024,
})

// ... 네트워크 문제를 재현해요 ...

const writtenPath = await netLog.stopLogging()
// writtenPath 의 파일을 이제 NetLog 뷰어로 열 수 있어요
```

<Callout type="warning">
`captureMode: 'includeSensitive'` 와 `'everything'` 은 **쿠키, 인증 헤더, 심지어 응답 본문까지** 평문 JSON 파일에 기록해요. 이렇게 캡처한 로그를 공개 버그 트래커에 절대 올리지 말고, 디버깅이 끝나면 바로 삭제하세요.
</Callout>

<Callout type="warning">
`stopLogging()` 을 한 번도 부르지 않으면 Electron 이 앱 종료 시 로그를 자동으로 flush 해요 — 하지만 파일이 언제 쓰이고 닫히는지 직접 제어하려면 본인이 멈추는 게 가장 좋아요.
</Callout>

<EasyNote>
`captureMode` 는 녹화의 "줌 배율" 이라고 생각하세요. `'default'` 는 요청이 _있었다_ 는 사실만 적고, `'everything'` 은 그 내용 전체를 기록해요. 많이 담을수록 파일이 커지고 — 민감해져요.
</EasyNote>

## 속성(Properties)

| 속성 | 타입 | 무엇인가 |
|---|---|---|
| `netLog.currentlyLogging` | `boolean` (읽기전용) | 기록 중이면 `true`, 아니면 `false` |

```js filename="guard.js"
const { netLog } = require('electron')

// 이미 돌고 있는 기록 위에 또 시작하지 않도록 방지
if (!netLog.currentlyLogging) {
  await netLog.startLogging('/tmp/net-log.json')
}
```

## 쉽게 풀어 설명

`netLog` 은 앱 인터넷 연결의 **블랙박스(주행 기록계)** 예요. 어떤 요청이 영문 모르게 실패하거나, 멈추거나, 엉뚱한 프록시로 갈 때, 기록을 켜고 → 문제를 재현하고 → 끕니다. 그러면 네트워크가 밟은 모든 단계를 순서대로 담은 파일 하나가 남아요.

이 모듈은 결국 세 가지 이름이 전부예요:

1. **`startLogging(path)`** — 녹화 시작. 얼마나 자세히 남길지 `captureMode` 로 정하고, 파일이 끝없이 커지지 않게 `maxFileSize` 를 줄 수 있어요.
2. **`stopLogging()`** — 녹화 정지. 모든 내용을 디스크에 flush 하고 파일 경로를 돌려줘요.
3. **`currentlyLogging`** — 작은 빨간 "녹화 중" 표시등. 첫 녹화 위에 두 번째 녹화를 시작하지 않게 해줘요.

남은 파일을 온라인 **NetLog 뷰어** 에 넣으면 무슨 일이 있었는지 그대로 재생할 수 있어요 — 어떤 호스트가 느렸는지, 리다이렉트가 어디로 갔는지, 왜 인증서가 거부됐는지. 단, 경고는 꼭 기억하세요: 더 자세한 캡처 모드는 비밀 정보를 기록하니, 그 파일은 비밀번호처럼 다뤄야 해요.
