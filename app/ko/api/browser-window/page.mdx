`BrowserWindow` 는 **Main 프로세스** 에서 `new` 로 만드는 **클래스** 예요. 인스턴스 하나가 실제 OS 창 하나이고, 그 안에는 웹 페이지(`WebContents`)가 들어 있어요.

```js filename="main.js"
const { BrowserWindow } = require('electron')

const win = new BrowserWindow({
  width: 800,
  height: 600,
  show: false, // 숨긴 채로 만들고, 페이지가 그려지면 보여줘요
})

win.once('ready-to-show', () => win.show())
win.loadFile('index.html')
```

<EasyNote>
`BrowserWindow` 는 **액자**, 그 안의 웹 페이지는 **그림** 이에요. 액자는 크기·위치·최대화/최소화 같은 걸 다루고, 그림(`win.webContents`)은 무엇이 그려질지를 다뤄요. 서로 다른 두 가지 일, 그래서 서로 다른 두 객체예요.
</EasyNote>

## 정적 메서드(Static methods)

인스턴스가 아니라 클래스 자체에 대고 호출해요.

| 메서드 | 하는 일 |
|---|---|
| `BrowserWindow.getAllWindows()` | 열려 있는 모든 창을 `BrowserWindow[]` 로 |
| `BrowserWindow.getFocusedWindow()` | 현재 포커스된 창, 없으면 `null` |
| `BrowserWindow.fromWebContents(webContents)` | 주어진 `WebContents` 를 가진 창, 없으면 `null` |
| `BrowserWindow.fromBrowserView(browserView)` _(Deprecated)_ | `BrowserView` 에 붙은 창, 없으면 `null` |
| `BrowserWindow.fromId(id)` | 주어진 숫자 `id` 의 창, 없으면 `null` |

```js filename="reopen.js"
// macOS: Dock 아이콘을 눌렀는데 열린 창이 없으면 다시 만들어요
app.on('activate', () => {
  if (BrowserWindow.getAllWindows().length === 0) createWindow()
})
```

## 인스턴스 이벤트(Instance events)

`win.on('이벤트명', 핸들러)` 로 듣습니다. _(macOS)_ / _(Windows)_ / _(Linux)_ 표시는 해당 플랫폼에서만 발생해요.

### 생명주기 & 표시

| 이벤트 | 발생 시점 |
|---|---|
| `page-title-updated` | 문서 제목이 바뀜 — `preventDefault()` 로 네이티브 제목 유지 |
| `close` | 창이 곧 닫히려 함 — `preventDefault()` 로 취소 가능 |
| `closed` | 창이 닫힘 — 참조를 버리세요, 이미 사라졌어요 |
| `ready-to-show` | 페이지가 그려져 깜빡임 없이 보여줄 수 있음 |
| `show` | 창이 보여짐 |
| `hide` | 창이 숨겨짐 |
| `focus` | 창이 포커스를 얻음 |
| `blur` | 창이 포커스를 잃음 |
| `query-session-end` _(Windows)_ | OS 세션이 곧 끝나려 함 — `preventDefault()` 로 지연 |
| `session-end` _(Windows)_ | OS 세션이 끝나는 중 (로그아웃/종료) |

```js filename="confirm-close.js"
win.on('close', (event) => {
  if (hasUnsavedWork()) {
    event.preventDefault() // 창을 열어둬요
    showSaveDialog()
  }
})

win.on('closed', () => {
  // 창이 사라졌어요 — GC 될 수 있도록 참조를 비워요
  win = null
})
```

<Callout type="warning">
`closed` 가 발생하면 창은 파괴돼요 — 이후 어떤 메서드를 호출해도 예외가 납니다. `closed` 핸들러 안에서 항상 참조를 `null` 로 비우세요.
</Callout>

### 응답성

| 이벤트 | 발생 시점 |
|---|---|
| `unresponsive` | 웹 페이지가 응답을 멈춤 (오래 도는 스크립트) |
| `responsive` | 웹 페이지가 다시 응답함 |

### 크기 · 위치 · 상태

| 이벤트 | 발생 시점 |
|---|---|
| `maximize` | 창이 최대화됨 |
| `unmaximize` | 최대화 상태에서 벗어남 |
| `minimize` | 창이 최소화됨 |
| `restore` | 최소화에서 복원됨 |
| `will-resize` _(macOS/Windows)_ | 리사이즈 직전 — `preventDefault()` 로 막기 |
| `resize` | 리사이즈 후 |
| `resized` _(macOS/Windows)_ | 리사이즈 동작이 끝남 |
| `will-move` _(macOS/Windows)_ | 이동 직전 — `preventDefault()` 로 막기 |
| `move` | 창이 이동 중 |
| `moved` _(macOS/Windows)_ | 이동 동작이 끝남 |
| `always-on-top-changed` | 항상 위 플래그가 토글됨 |

### 전체화면

| 이벤트 | 발생 시점 |
|---|---|
| `enter-full-screen` | 전체화면 진입 |
| `leave-full-screen` | 전체화면 해제 |
| `enter-html-full-screen` | 페이지 요소가 HTML 전체화면 진입 |
| `leave-html-full-screen` | 페이지 요소가 HTML 전체화면 해제 |

```js filename="show-when-ready.js"
const win = new BrowserWindow({ show: false })

// 흰 깜빡임 방지: 페이지가 그려진 뒤에만 창을 보여줘요
win.once('ready-to-show', () => {
  win.show()
})
```

### 제스처 & 플랫폼 추가

| 이벤트 | 발생 시점 |
|---|---|
| `app-command` _(Windows/Linux)_ | App Command 호출됨 (예: 미디어 키, 브라우저 뒤로) |
| `swipe` _(macOS)_ | 트랙패드 세 손가락 스와이프 |
| `rotate-gesture` _(macOS)_ | 트랙패드 회전 제스처 |
| `sheet-begin` _(macOS)_ | 창 시트가 열림 |
| `sheet-end` _(macOS)_ | 창 시트가 닫힘 |
| `new-window-for-tab` _(macOS)_ | 네이티브 "새 탭" 버튼 클릭 |
| `system-context-menu` _(Windows/Linux)_ | 시스템 컨텍스트 메뉴 트리거 — `preventDefault()` 로 억제 |

<EasyNote>
`ready-to-show` 는 거의 모든 앱이 쓰는 이벤트예요. 창을 `show: false` 로 만든 뒤 `ready-to-show` 에서 보여주면, 페이지가 로드되는 동안 빈 흰 사각형을 사용자가 보지 않아요.
</EasyNote>

<Callout type="warning">
`BrowserView` 관련 메서드 — `win.setBrowserView`, `win.getBrowserView`, `win.addBrowserView`, `win.removeBrowserView`, `win.setTopBrowserView`, `win.getBrowserViews` — 는 **사용 중단(deprecated)** 됐어요. 새 코드에서는 `WebContentsView` 와 `BaseWindow` 를 쓰세요.
</Callout>

## 쉽게 풀어 설명

`BrowserWindow` 는 이름 그대로예요: **안에 웹 브라우저가 들어 있는 창**. `new BrowserWindow(options)` 로 하나 만들면, 그때부터 생각할 게 두 가지로 나뉘어요.

1. **액자** — `BrowserWindow` 자신이에요. 어디에 있지(`setBounds`)? 얼마나 크지(`setSize`)? 최대화·최소화·전체화면·항상 위 상태인가? 사용자가 리사이즈하거나 닫을 수 있나? 이 모든 게 `win` 의 메서드나 속성이에요.
2. **안의 그림** — `win.webContents`, 즉 실제 HTML/CSS/JS 페이지예요. `loadFile` 나 `loadURL` 로 불러오고, _무엇이 그려질지_ 에 관한 모든 건 여기 있어요.

거의 모든 앱이 쓰는 패턴은 이거예요: 창을 숨긴 채(`show: false`) 만들고, 페이지를 로드한 뒤, **`ready-to-show`** 에서 보여줘서 흰 깜빡임을 없애는 것. 그다음 저장 안 한 작업이 있으면 **`close`** 를 처리하고, **`closed`** 에서 참조를 비워 파괴된 창을 다시 건드리지 않게 해요.

이름 세 개만 기억하면 나머지는 찾아보면 돼요: **`loadFile`**(페이지 넣기), **`ready-to-show`**(깔끔하게 보여주기), **`win.webContents`**(페이지와 대화하기). 이게 모든 Electron 창의 핵심이에요.
