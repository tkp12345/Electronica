`NotificationResponse` 객체는 사용자가 알림에 어떻게 반응했는지를 설명합니다. macOS에서 `app.on('activate', ...)`에 등록한 콜백으로, `notification` 활성화 타입을 통해 전달됩니다.

## 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `actionIdentifier` | `string` | 사용자가 선택한 액션의 식별자 문자열입니다. |
| `date` | `number` | 알림이 전달된 날짜입니다. |
| `identifier` | `string` | 이 알림 요청의 고유 식별자입니다. |
| `userInfo` | `Record<string, any>` | 알림에 연결된 사용자 지정 정보를 담은 딕셔너리입니다. |
| `userText` | `string` | _(선택)_ 사용자가 입력하거나 선택한 텍스트입니다. |

```js
function onNotificationResponse(response) {
  if (response.actionIdentifier === 'reply') {
    console.log('사용자가 답장함:', response.userText)
  }
}
```

<EasyNote>
NotificationResponse는 누군가에게 쪽지를 건넨 뒤 되돌아오는 답장 쪽지와 같습니다. 어떤 버튼을 눌렀는지, 그리고 입력란이 있었다면 무엇을 적었는지 알려줍니다.
</EasyNote>

## 쉽게 풀어 설명

앱이 버튼이나 답장 입력란이 있는 알림을 띄우면, 시스템은 사용자가 무엇을 했는지 앱에 알려줄 방법이 필요합니다. `NotificationResponse` 객체가 바로 그 보고서입니다. 사용자가 어떤 액션을 골랐는지, 어떤 텍스트를 입력했는지, 알림이 언제 전달되었는지, 그리고 처음에 알림에 붙여 두었던 추가 데이터까지 담고 있습니다. 활성화 콜백에서 이 객체를 읽어 사용자의 선택에 맞게 앱이 동작하도록 만들면 됩니다.
