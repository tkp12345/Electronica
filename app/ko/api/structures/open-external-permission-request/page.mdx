`OpenExternalPermissionRequest` 객체는 페이지가 외부 URL을 열려고 할 때(예: `shell.openExternal` 또는 앱을 벗어나는 링크) 권한 핸들러로 전달됩니다. 기본 `PermissionRequest`를 확장하므로 표준 요청 정보와 함께 요청된 외부 URL을 담고 있습니다. 이 객체는 `session`의 `setPermissionRequestHandler`와 `setPermissionCheckHandler` 안에서 전달받게 됩니다.

## 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `externalURL` | `string` | _(선택)_ `openExternal` 요청의 URL입니다. |
| `requestingUrl` | `string` | 요청하는 프레임이 마지막으로 로드한 URL입니다. |
| `isMainFrame` | `boolean` | 요청을 보내는 프레임이 메인 프레임인지 여부입니다. |

```js
const { session } = require('electron')

session.defaultSession.setPermissionRequestHandler((webContents, permission, callback, details) => {
  if (permission === 'openExternal') {
    // details 는 OpenExternalPermissionRequest 입니다
    const allowed = details.externalURL?.startsWith('https://example.com')
    return callback(Boolean(allowed))
  }
  callback(true)
})
```

## 쉽게 말하면

앱 안의 웹 페이지가 링크를 사용자의 실제 브라우저나 다른 앱에서 열려고 하면, Electron이 먼저 여러분에게 허락을 구하면서 이 객체를 건네줍니다. 이 객체로 외부 URL을 확인하고 허용할지 결정할 수 있어요. 사용자가 여러분의 승인 없이 의심스러운 사이트로 이동하는 것을 막는 간단한 방법입니다.
