`InputEvent`는 Electron의 저수준 입력 이벤트가 공통으로 가지는 기본 객체입니다. `webContents.sendInputEvent()`로 가상 입력을 보낼 때 사용하며, `MouseInputEvent`나 `KeyboardInputEvent` 같은 더 구체적인 이벤트들이 이 객체를 확장하는 토대가 됩니다.

## 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `type` | `string` | 이벤트의 종류입니다. 예를 들어 `mouseDown`, `mouseUp`, `mouseMove`, `mouseEnter`, `mouseLeave`, `contextMenu`, `mouseWheel`, `rawKeyDown`, `keyDown`, `keyUp`, `char`, 제스처 타입(`gestureScrollBegin`, `gesturePinchUpdate`, ...), 터치 이벤트(`touchStart`, `touchMove`, ...), 포인터 이벤트(`pointerDown`, `pointerUp`, ...) 등이 있습니다. |
| `modifiers` | `string[]` | _(선택)_ 이벤트가 발생할 때 함께 눌려 있던 조합 키/버튼을 나타냅니다. `shift`, `control`, `ctrl`, `alt`, `meta`, `command`, `cmd`, `iskeypad`, `isautorepeat`, `leftbuttondown`, `middlebuttondown`, `rightbuttondown`, `capslock`, `numlock`, `left`, `right` 등의 값을 사용할 수 있습니다. |

```js
const inputEvent = {
  type: 'keyDown',
  modifiers: ['shift', 'ctrl'],
}
```

<EasyNote>
`InputEvent`는 모든 입력 메시지가 담겨 오는 봉투라고 생각하면 됩니다. 항상 어떤 종류의 이벤트인지 알려 주고, 그때 어떤 키가 함께 눌려 있었는지를 선택적으로 적어 둡니다.
</EasyNote>

## 쉽게 풀어 설명

대부분의 경우 `InputEvent`를 직접 만들 일은 없습니다. Electron이 건네주는 키보드·마우스·터치 이벤트가 모두 이 형태에서 출발하기 때문에, 언제나 `type`을 가지고 있고 필요에 따라 `modifiers` 목록을 함께 담고 있습니다. 직접 입력을 흉내 내고 싶을 때는 이 필드들을 채워서 `webContents.sendInputEvent()`에 넘기면, 실제로 키를 누르거나 클릭한 것처럼 페이지가 반응하게 만들 수 있습니다.
