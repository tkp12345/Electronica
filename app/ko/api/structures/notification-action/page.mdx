`NotificationAction` 객체는 알림에 붙는 하나의 상호작용 컨트롤을 나타냅니다. `Notification`을 생성할 때 `actions` 옵션에 이 객체들의 배열을 전달하며, 사용자가 그중 하나를 클릭하면 알림의 `action` 이벤트가 발생합니다.

## 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `type` | `string` | 동작의 종류입니다. `button` 또는 `selection` 중 하나입니다. `selection` 타입은 _(Windows)_ 전용이며 드롭다운 콤보 박스로 표시됩니다. |
| `text` | `string` | _(선택)_ 동작에 표시되는 라벨입니다. macOS에서는 생략 시 첫 번째 버튼이 지역화된 "Show"를 기본값으로 사용합니다. |

```js
const { Notification } = require('electron')

const notification = new Notification({
  title: '다운로드 완료',
  body: '파일이 준비되었습니다.',
  actions: [
    { type: 'button', text: '열기' },
    { type: 'button', text: '폴더에서 보기' }
  ]
})

notification.on('action', (event, index) => {
  // index로 어떤 동작 버튼이 클릭되었는지 식별합니다
})
```

<Callout type="info">
**플랫폼 참고.** macOS에서는 버튼이 나타나려면 앱이 서명되어 있어야 하고 Info.plist의 `NSUserNotificationAlertStyle`이 `alert`로 설정되어 있어야 합니다. 또한 첫 번째 버튼만 바로 보이고 나머지는 마우스를 올렸을 때 나타나며, `hasReply`가 설정되면 버튼은 무시됩니다. Windows에서는 `selection` 동작에 옵션 라벨이 함께 필요하고, `action` 이벤트가 `actionIndex`와 0부터 시작하는 `selectedIndex`(사용할 수 없으면 `-1`)를 함께 전달합니다.
</Callout>

<EasyNote>
NotificationAction은 팝업 알림 한쪽에 인쇄된 버튼과 같습니다. 누르면 앱이 사용자가 어떤 선택을 했는지 정확히 알게 됩니다.
</EasyNote>

## 쉽게 풀어 설명

NotificationAction을 쓰면 알림이 단순히 글자만 보여주는 것을 넘어, 팝업 안에 클릭할 수 있는 버튼(또는 Windows에서는 드롭다운)을 바로 추가할 수 있습니다. "열기", "다시 알림", "답장" 같은 빠른 선택지를 제공해서, 사용자가 앱으로 다시 돌아가지 않고도 바로 동작하게 만들 때 사용합니다. 사용자가 하나를 고르면 알림의 `action` 이벤트를 통해 코드가 그 사실을 전달받습니다.
