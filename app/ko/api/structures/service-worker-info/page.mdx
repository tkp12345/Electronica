이 구조체는 세션에 현재 등록되어 실행 중인 서비스 워커를 설명합니다. `Session`의 `serviceWorkers` 객체를 통해 얻을 수 있습니다(예: `session.defaultSession.serviceWorkers.getInfoFromVersionID()`).

## 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `scriptUrl` | `string` | 이 서비스 워커가 실행하는 스크립트의 전체 URL입니다. |
| `scope` | `string` | 이 서비스 워커가 활성화되는 기준 URL입니다. |
| `renderProcessId` | `number` | 이 서비스 워커가 실행되는 프로세스의 가상 ID입니다. OS 수준의 PID가 아니며, `webContents.getProcessId()`에서 사용하는 ID와 일치합니다. |
| `versionId` | `number` | 서비스 워커 버전의 ID입니다. |

```js
{
  scriptUrl: 'https://example.com/sw.js',
  scope: 'https://example.com/',
  renderProcessId: 12,
  versionId: 3
}
```

<EasyNote>
백그라운드에서 동작하는 도우미 스크립트 하나의 신분증이라고 생각하면 됩니다. 어떤 파일인지, 사이트의 어느 영역을 담당하는지, 어디서 실행되고 있는지를 알려줍니다.
</EasyNote>

## 쉽게 말하면

서비스 워커는 캐싱이나 오프라인 지원 같은 일을 처리하기 위해 백그라운드에서 실행되는 작은 스크립트입니다. 이 객체는 그런 스크립트 하나에 대한 성적표와 같아서, 원본 파일이 무엇이고 어떤 페이지를 제어하며 어떤 프로세스가 호스팅하고 있는지를 알려줍니다. 세션에 연결된 서비스 워커를 점검하거나 디버깅할 때 사용하게 됩니다.
