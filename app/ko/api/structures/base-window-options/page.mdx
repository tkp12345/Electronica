`BaseWindowOptions`는 `BaseWindow` 생성자가 받는 일반 옵션 객체이며, `BrowserWindow`도 이를 상속합니다. 모든 필드는 선택 사항이고, 창이 생성될 때의 모습과 동작을 결정합니다.

## 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `width` | `Integer` | _(선택)_ 창의 너비(픽셀). 기본값은 `800`입니다. |
| `height` | `Integer` | _(선택)_ 창의 높이(픽셀). 기본값은 `600`입니다. |
| `x` | `Integer` | _(선택)_ 화면 기준 창의 왼쪽 오프셋. 기본값은 가운데 정렬입니다. |
| `y` | `Integer` | _(선택)_ 화면 기준 창의 위쪽 오프셋. 기본값은 가운데 정렬입니다. |
| `useContentSize` | `boolean` | _(선택)_ `width`/`height`를 전체 창이 아닌 웹 페이지 크기로 취급합니다. |
| `center` | `boolean` | _(선택)_ 창을 화면 가운데에 표시합니다. 기본값은 `false`입니다. |
| `minWidth` | `Integer` | _(선택)_ 창의 최소 너비. 기본값은 `0`입니다. |
| `minHeight` | `Integer` | _(선택)_ 창의 최소 높이. 기본값은 `0`입니다. |
| `maxWidth` | `Integer` | _(선택)_ 창의 최대 너비. 기본값은 제한 없음입니다. |
| `maxHeight` | `Integer` | _(선택)_ 창의 최대 높이. 기본값은 제한 없음입니다. |
| `resizable` | `boolean` | _(선택)_ 창 크기 조절 가능 여부. 기본값은 `true`입니다. |
| `movable` | `boolean` | _(선택)_ _(macOS / Windows)_ 창 이동 가능 여부. Linux에서는 구현되지 않았습니다. |
| `minimizable` | `boolean` | _(선택)_ _(macOS / Windows)_ 최소화 가능 여부. Linux에서는 구현되지 않았습니다. |
| `maximizable` | `boolean` | _(선택)_ _(macOS / Windows)_ 최대화 가능 여부. Linux에서는 구현되지 않았습니다. |
| `closable` | `boolean` | _(선택)_ _(macOS / Windows)_ 닫기 가능 여부. Linux에서는 구현되지 않았습니다. |
| `focusable` | `boolean` | _(선택)_ 창의 포커스 획득 가능 여부. Windows/Linux의 작업 표시줄 동작에 영향을 줍니다. |
| `alwaysOnTop` | `boolean` | _(선택)_ 창이 항상 다른 창 위에 표시될지 여부입니다. |
| `fullscreen` | `boolean` | _(선택)_ 생성 시 창을 전체 화면으로 표시합니다. |
| `fullscreenable` | `boolean` | _(선택)_ 창을 전체 화면 모드로 전환할 수 있는지 여부입니다. |
| `simpleFullscreen` | `boolean` | _(선택)_ _(macOS)_ Lion 이전 방식의 전체 화면을 사용합니다. 기본값은 `false`입니다. |
| `skipTaskbar` | `boolean` | _(선택)_ _(macOS / Windows)_ 작업 표시줄에 창을 표시할지 여부. 기본값은 `false`입니다. |
| `hiddenInMissionControl` | `boolean` | _(선택)_ _(macOS)_ 사용자가 미션 컨트롤로 전환할 때 창을 숨깁니다. |
| `kiosk` | `boolean` | _(선택)_ 창의 키오스크 모드 여부. 기본값은 `false`입니다. |
| `title` | `string` | _(선택)_ 기본 창 제목. 기본값은 `"Electron"`입니다. |
| `icon` | `NativeImage \| string` | _(선택)_ 창 아이콘. Windows에서는 ICO 형식을 권장합니다. |
| `show` | `boolean` | _(선택)_ 생성 시 창을 표시할지 여부. 기본값은 `true`입니다. |
| `frame` | `boolean` | _(선택)_ `false`로 설정하면 테두리 없는 창을 만듭니다. |
| `parent` | `BaseWindow` | _(선택)_ 부모 창 참조입니다. |
| `modal` | `boolean` | _(선택)_ 모달 창 여부. 자식 창일 때만 동작합니다. |
| `acceptFirstMouse` | `boolean` | _(선택)_ _(macOS)_ 비활성 창을 클릭하면 그 클릭이 웹 콘텐츠로 전달됩니다. |
| `disableAutoHideCursor` | `boolean` | _(선택)_ 타이핑할 때 커서를 숨길지 여부. 기본값은 `false`입니다. |
| `autoHideMenuBar` | `boolean` | _(선택)_ _(Linux / Windows)_ Alt 키를 누르지 않는 한 메뉴 바를 숨깁니다. |
| `enableLargerThanScreen` | `boolean` | _(선택)_ _(macOS)_ 창을 화면보다 크게 조절할 수 있게 합니다. |
| `backgroundColor` | `string` | _(선택)_ 배경색. Hex, RGB, RGBA, HSL, HSLA 또는 CSS 색상 이름을 사용합니다. |
| `hasShadow` | `boolean` | _(선택)_ 창에 그림자를 표시할지 여부. 기본값은 `true`입니다. |
| `opacity` | `number` | _(선택)_ _(macOS / Windows)_ 초기 창 불투명도. `0.0`에서 `1.0` 사이입니다. |
| `darkTheme` | `boolean` | _(선택)_ 창에 다크 테마를 강제합니다. 일부 GTK+3 데스크톱 환경에서만 동작합니다. |
| `transparent` | `boolean` | _(선택)_ 창을 투명하게 만듭니다. Windows에서는 테두리 없는 창이 필요합니다. |
| `type` | `string` | _(선택)_ 창 유형 분류(`desktop`, `dock`, `panel`, `toolbar` 등)입니다. |
| `visualEffectState` | `string` | _(선택)_ _(macOS)_ 배경 효과 상태: `followWindow`, `active`, `inactive`. |
| `titleBarStyle` | `string` | _(선택)_ 제목 표시줄 렌더링: `default`, `hidden`, `hiddenInset`, `customButtonsOnHover`. |
| `titleBarOverlay` | `Object \| boolean` | _(선택)_ 창 컨트롤 오버레이 설정. 중첩 필드: `color`(`string`, Windows/Linux), `symbolColor`(`string`, Windows/Linux), `height`(`Integer`). |
| `accentColor` | `boolean \| string` | _(선택)_ _(Windows)_ 창 강조색. Hex, RGB, RGBA, HSL, HSLA 또는 CSS 색상 이름을 사용합니다. |
| `trafficLightPosition` | `Point` | _(선택)_ _(macOS)_ 테두리 없는 창에서 신호등 버튼의 사용자 지정 위치입니다. |
| `roundedCorners` | `boolean` | _(선택)_ _(macOS / Windows)_ 테두리 없는 창의 모서리를 둥글게 할지 여부입니다. |
| `thickFrame` | `boolean` | _(선택)_ _(Windows)_ 테두리 없는 창에 `WS_THICKFRAME`을 사용합니다. 크기 조절 동작에 영향을 줍니다. |
| `vibrancy` | `string` | _(선택)_ _(macOS)_ 비브런시 효과 유형(`appearance-based`, `titlebar`, `selection`, `menu` 등)입니다. |
| `backgroundMaterial` | `string` | _(선택)_ _(Windows)_ 시스템이 그리는 배경: `auto`, `none`, `mica`, `acrylic`, `tabbed`. |
| `zoomToPageWidth` | `boolean` | _(선택)_ _(macOS)_ 초록 버튼을 Option-클릭하면 웹 페이지 너비에 맞춰 확대합니다. |
| `tabbingIdentifier` | `string` | _(선택)_ _(macOS)_ 네이티브 탭 그룹 이름. `new-window-for-tab` 이벤트를 발생시킵니다. |

```js
const { BaseWindow } = require('electron')

const win = new BaseWindow({
  width: 1000,
  height: 700,
  center: true,
  backgroundColor: '#1e1e1e',
  titleBarStyle: 'hidden'
})
```

<EasyNote>
새 창을 요청할 때 작성하는 주문서라고 생각하시면 됩니다. 크기, 위치, 색상, 그리고 어떤 버튼을 둘지를 적어 넣는 셈입니다.
</EasyNote>

## 쉽게 풀어 설명

앱이 창을 열 때, 원하는 모습을 설명하기 위해 이 객체를 Electron에 전달합니다. `width`와 `height`로 크기를 정하고, `center`나 `x`/`y`로 위치를 결정하며, `resizable`이나 `alwaysOnTop` 같은 플래그로 동작 방식을 정합니다. 많은 필드가 플랫폼별로 동작하므로, `vibrancy` 같은 설정은 macOS에서만 의미가 있습니다. 창을 만들 때는 거의 항상 `BaseWindowOptions`(또는 `BrowserWindow`가 쓰는 더 풍부한 버전)를 전달하게 됩니다.
