`MemoryInfo`는 단일 프로세스의 메모리 사용량을 설명하는 평범한 객체입니다. `process.getProcessMemoryInfo()`가 반환하며, `app.getAppMetrics()` 같은 리포트 안에서도 사용됩니다. 모든 값은 킬로바이트(KB) 단위로 보고됩니다.

## 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `workingSetSize` | `Integer` | 현재 실제 물리 RAM에 고정되어 있는 메모리 양입니다 (KB 단위). |
| `peakWorkingSetSize` | `Integer` | 지금까지 실제 물리 RAM에 고정되었던 최대 메모리 양입니다 (KB 단위). |
| `privateBytes` | `Integer` | _(선택)_ _(Windows)_ JS 힙이나 HTML 콘텐츠처럼 다른 프로세스와 공유되지 않는 메모리 양입니다 (KB 단위). |

```js
// 일반적인 MemoryInfo 객체
const memory = {
  workingSetSize: 32768,
  peakWorkingSetSize: 65536,
  privateBytes: 16384 // Windows 전용
}
```

<EasyNote>
하나의 프로세스를 위한 연료 게이지라고 생각하시면 됩니다. 지금 얼마나 많은 RAM을 쓰고 있는지, 그리고 지금까지 도달했던 최고치가 얼마인지 알려줍니다.
</EasyNote>

## 쉽게 풀어 설명

앱(또는 앱의 특정 프로세스)이 메모리를 얼마나 쓰고 있는지 알고 싶을 때 `MemoryInfo`를 사용하시면 됩니다. 메모리 누수를 찾거나, 진단 패널을 만들거나, 시간에 따른 자원 사용량을 기록할 때 유용합니다. 두 개의 주요 값은 현재 사용량과 역대 최고치를 알려주고, `privateBytes`(Windows 전용)는 그 프로세스만 단독으로 차지하는 메모리를 보여줍니다.
