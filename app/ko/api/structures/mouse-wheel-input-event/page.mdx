`MouseWheelInputEvent`는 페이지 안에서 스크롤을 흉내 내고 싶을 때 `webContents.sendInputEvent()`에 전달하는 객체입니다. `MouseInputEvent`(다시 `InputEvent`를 확장)를 확장하므로, 포인터 위치와 수정자 키 필드에 더해 휠 전용 델타 값을 함께 가집니다.

## 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `type` | `string` | 이벤트의 종류로, 휠 이벤트일 때는 반드시 `mouseWheel`이어야 합니다. |
| `modifiers` | `string[]` | _(선택)_ 이벤트 중에 눌린 수정자 키입니다. 예: `shift`, `control`, `alt`, `meta`, `command` 및 하드웨어/상태 플래그. |
| `x` | `Integer` | 창의 콘텐츠 영역 기준 포인터의 가로 좌표입니다. |
| `y` | `Integer` | 창의 콘텐츠 영역 기준 포인터의 세로 좌표입니다. |
| `button` | `string` | _(선택)_ 눌린 버튼으로, `left`, `middle`, `right` 중 하나입니다. |
| `globalX` | `Integer` | _(선택)_ 전역(화면) 기준 가로 좌표입니다. |
| `globalY` | `Integer` | _(선택)_ 전역(화면) 기준 세로 좌표입니다. |
| `movementX` | `Integer` | _(선택)_ 직전 마우스 이벤트 이후의 가로 이동량입니다. |
| `movementY` | `Integer` | _(선택)_ 직전 마우스 이벤트 이후의 세로 이동량입니다. |
| `clickCount` | `Integer` | _(선택)_ 이 이벤트의 연속 클릭 횟수입니다. |
| `deltaX` | `Integer` | _(선택)_ 가로로 스크롤된 양입니다. |
| `deltaY` | `Integer` | _(선택)_ 세로로 스크롤된 양입니다. |
| `wheelTicksX` | `Integer` | _(선택)_ 가로 휠 틱(눈금) 수입니다. |
| `wheelTicksY` | `Integer` | _(선택)_ 세로 휠 틱(눈금) 수입니다. |
| `accelerationRatioX` | `Integer` | _(선택)_ 가로 스크롤 가속 비율입니다. |
| `accelerationRatioY` | `Integer` | _(선택)_ 세로 스크롤 가속 비율입니다. |
| `hasPreciseScrollingDeltas` | `boolean` | _(선택)_ 델타 값이 트랙패드 같은 정밀 입력 장치에서 온 것인지 여부입니다. |
| `canScroll` | `boolean` | _(선택)_ 이 이벤트가 스크롤을 일으킬 수 있는지 여부입니다. |

```js
// (200, 200) 지점에서 페이지를 약 120px 아래로 스크롤합니다
webContents.sendInputEvent({
  type: 'mouseWheel',
  x: 200,
  y: 200,
  deltaY: -120,
  canScroll: true,
})
```

<EasyNote>
"여기 이 위치에서 사용자가 스크롤 휠을 이만큼 돌렸다고 치자"라고 적힌 쪽지를 페이지에 건네주는 것이라고 생각하면 됩니다.
</EasyNote>

## 쉽게 풀어 설명

실제 마우스 없이 앱이 스스로 페이지를 스크롤하게 만들고 싶을 때 이 객체를 사용합니다 — 자동화 테스트, 키오스크 데모, 접근성 도우미 등에 유용합니다. 포인터 위치(`x`, `y`)와 얼마나 스크롤할지(`deltaY`는 위아래, `deltaX`는 좌우)를 지정한 뒤 `webContents.sendInputEvent()`로 보내면 됩니다. 나머지 필드들은 트랙패드처럼 부드럽고 정밀한 스크롤 느낌을 흉내 내는 등 세밀한 조정을 돕습니다.
