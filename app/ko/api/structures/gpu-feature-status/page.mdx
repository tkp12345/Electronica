`GPUFeatureStatus`는 사용자 컴퓨터에서 각 그래픽 기능의 상태를 설명하는 일반 객체입니다. `app.getGPUFeatureStatus()`가 반환하는 값입니다. 모든 필드는 문자열이며, 그 값은 `enabled`, `disabled`, `unavailable_software`, `disabled_software`처럼 문서에 정의된 상태 중 하나입니다.

## 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `2d_canvas` | `string` | 캔버스. |
| `flash_3d` | `string` | 플래시. |
| `flash_stage3d` | `string` | 플래시 Stage3D. |
| `flash_stage3d_baseline` | `string` | 플래시 Stage3D 기본 프로파일. |
| `gpu_compositing` | `string` | 합성(Compositing). |
| `multiple_raster_threads` | `string` | 다중 래스터 스레드. |
| `native_gpu_memory_buffers` | `string` | 네이티브 GpuMemoryBuffers. |
| `rasterization` | `string` | 래스터화(Rasterization). |
| `video_decode` | `string` | 비디오 디코드. |
| `video_encode` | `string` | 비디오 인코드. |
| `vpx_decode` | `string` | VPx 비디오 디코드. |
| `webgl` | `string` | WebGL. |
| `webgl2` | `string` | WebGL2. |

```js
const { app } = require('electron')

app.whenReady().then(() => {
  const status = app.getGPUFeatureStatus()
  // {
  //   '2d_canvas': 'enabled',
  //   gpu_compositing: 'enabled',
  //   webgl: 'enabled',
  //   ...
  // }
})
```

## 쉽게 말하면

그래픽이 느리거나 이상하게 보일 때, GPU가 실제로 일을 돕고 있는지 알고 싶다면 `GPUFeatureStatus`를 사용합니다. `webgl`이나 `gpu_compositing` 같은 기능이 `disabled` 또는 `software` 폴백 상태로 나오면, 그 작업을 그래픽 카드가 아니라 CPU가 대신 처리하고 있다는 뜻이며, 이는 화면 끊김이나 높은 전력 사용의 원인이 될 수 있습니다. 이 객체를 기록해 두면 여러 기기에서 렌더링 문제를 빠르게 디버깅할 수 있습니다.
