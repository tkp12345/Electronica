`DesktopCapturerSource`는 캡처할 수 있는 하나의 대상을 설명합니다. 전체 화면일 수도 있고 개별 애플리케이션 창일 수도 있습니다. `desktopCapturer.getSources()` 메서드가 이 객체들의 배열을 반환하므로, 사용자가 녹화하거나 공유할 대상을 직접 고를 수 있습니다.

## 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `id` | `string` | 창 또는 화면의 식별자로, `chromeMediaSourceId` 제약 조건으로 사용할 수 있습니다. 형식은 `window:XX:YY` 또는 `screen:ZZ:0` 입니다. |
| `name` | `string` | 설명용 이름입니다. 화면 소스는 `Entire Screen` 또는 `Screen <index>`로 이름이 지정되고, 창 소스는 해당 창의 제목과 일치합니다. |
| `thumbnail` | `NativeImage` | 소스의 미리보기 이미지입니다. 실제 크기는 화면/창의 배율에 따라 달라지며, 요청한 크기와 일치한다고 보장되지 않습니다. |
| `display_id` | `string` | Screen API가 반환하는 `Display`의 `id`와 대응되는 고유 식별자입니다. 사용할 수 없는 경우 빈 값일 수 있습니다. |
| `appIcon` | `NativeImage` | _(선택)_ 창 소스의 애플리케이션 아이콘이며, 화면 소스의 경우 `null`입니다. 아이콘 크기는 애플리케이션이 제공하는 데이터에 따라 달라집니다. |

```js
const { desktopCapturer } = require('electron')

const sources = await desktopCapturer.getSources({ types: ['window', 'screen'] })
const first = sources[0]
// first => { id: 'screen:0:0', name: 'Entire Screen', thumbnail, display_id, appIcon }
```

<EasyNote>
"무엇을 공유할까요?" 메뉴의 한 줄이라고 생각하면 됩니다. 각 항목은 이름과 미리보기 썸네일, 그리고 시스템이 캡처를 시작할 때 쓰는 숨겨진 ID를 가지고 있습니다.
</EasyNote>

## 쉽게 풀어 설명

화면 공유나 화면 녹화 기능을 만들 때는 사용자가 캡처할 수 있는 대상 목록을 보여줘야 합니다. 각 `DesktopCapturerSource`는 그 목록의 한 항목입니다. 보기 좋은 `name`, 미리보기로 그려줄 `thumbnail`, 그리고 실제로 픽셀을 가져오기 위해 미디어 API에 전달하는 `id`를 담고 있습니다. 이 객체는 직접 만들기보다는 주로 읽어서 사용하게 됩니다.
