`Transaction` 객체는 macOS App Store의 인앱 결제 거래 하나를 설명합니다. 결제 상태가 바뀔 때 `inAppPurchase` 모듈의 `transactions-updated` 이벤트를 통해 `Transaction` 객체를 받게 됩니다.

## 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `transactionIdentifier` | `string` | 성공한 결제 거래를 고유하게 식별하는 문자열입니다. |
| `transactionDate` | `string` | 해당 거래가 App Store 결제 대기열에 추가된 날짜입니다. |
| `originalTransactionIdentifier` | `string` | App Store가 복원한 원본 거래의 식별자입니다. |
| `transactionState` | `string` | 현재 상태입니다: `purchasing`, `purchased`, `failed`, `restored`, `deferred`. |
| `errorCode` | `Integer` | 거래를 처리하는 중 오류가 발생했을 때의 오류 코드입니다. |
| `errorMessage` | `string` | 거래를 처리하는 중 오류가 발생했을 때의 오류 메시지입니다. |
| `payment` | `Object` | 이 거래의 결제 정보입니다 (아래 중첩 필드 참고). |
| `payment.productIdentifier` | `string` | 구매한 상품의 식별자입니다. |
| `payment.quantity` | `Integer` | 구매 수량입니다. |
| `payment.applicationUsername` | `string` | 여러분의 시스템에서 사용자 계정을 가리키는 불투명한(opaque) 식별자입니다. |
| `payment.paymentDiscount` | `PaymentDiscount` | _(선택)_ 결제에 적용할 할인 혜택의 세부 정보입니다. |

```js
const { inAppPurchase } = require('electron')

inAppPurchase.on('transactions-updated', (event, transactions) => {
  for (const transaction of transactions) {
    if (transaction.transactionState === 'purchased') {
      // 상품을 지급한 뒤 거래를 마무리합니다.
      inAppPurchase.finishTransactionByDate(transaction.transactionDate)
    }
  }
})
```

<EasyNote>
Transaction은 인앱 구매의 영수증과 같습니다. 무엇을, 언제, 누가 샀는지 그리고 결제가 정상적으로 처리됐는지를 보여 줍니다.
</EasyNote>

## 쉽게 풀어 설명

누군가 macOS 앱 안에서 App Store를 통해 무언가를 구매하면, Electron은 그 구매가 어떻게 진행되는지 알려 주기 위해 `Transaction`을 건네줍니다. `transactionState`를 확인해 성공·실패·대기 중인지 보고, `payment`를 읽어 어떤 상품을 몇 개 샀는지 파악하며, 식별자들을 이용해 상품을 지급하고 거래를 완료 처리하면 됩니다.
