`USBDevice` 객체는 WebUSB API를 통해 앱이 통신할 수 있는 하나의 USB 하드웨어 장치를 나타냅니다. 이 객체는 [`session`](/ko/api/session) 모듈의 `select-usb-device` 이벤트와 USB 장치 권한 핸들러에 전달되므로, 앱이 어떤 장치를 사용할 수 있는지 결정할 수 있습니다.

## 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `deviceId` | `string` | 장치의 고유 식별자입니다. |
| `vendorId` | `Integer` | USB 공급업체(vendor) ID입니다. |
| `productId` | `Integer` | USB 제품(product) ID입니다. |
| `productName` | `string` | 장치의 이름입니다. _(선택)_ |
| `serialNumber` | `string` | USB 장치의 일련번호입니다. _(선택)_ |
| `manufacturerName` | `string` | 장치의 제조사 이름입니다. _(선택)_ |
| `usbVersionMajor` | `Integer` | 장치가 지원하는 USB 프로토콜의 메이저 버전입니다. |
| `usbVersionMinor` | `Integer` | 장치가 지원하는 USB 프로토콜의 마이너 버전입니다. |
| `usbVersionSubminor` | `Integer` | 장치가 지원하는 USB 프로토콜의 서브마이너 버전입니다. |
| `deviceClass` | `Integer` | 장치가 지원하는 통신 인터페이스의 장치 클래스입니다. |
| `deviceSubclass` | `Integer` | 장치가 지원하는 통신 인터페이스의 장치 서브클래스입니다. |
| `deviceProtocol` | `Integer` | 장치가 지원하는 통신 인터페이스의 장치 프로토콜입니다. |
| `deviceVersionMajor` | `Integer` | 제조사가 정의한 장치의 메이저 버전 번호입니다. |
| `deviceVersionMinor` | `Integer` | 제조사가 정의한 장치의 마이너 버전 번호입니다. |
| `deviceVersionSubminor` | `Integer` | 제조사가 정의한 장치의 서브마이너 버전 번호입니다. |
| `configuration` | `Object` | 현재 선택된 구성을 설명하는 USBConfiguration 객체입니다. 각 객체는 `configurationValue` (`Integer`), `configurationName` (`string`), `interfaces` (`Object[]`)를 가집니다. _(선택)_ |
| `configurations` | `Object[]` | 페어링된 USB 장치를 제어하기 위한 USBConfiguration 객체의 배열입니다. |

```js
const { app } = require('electron')

app.whenReady().then(() => {
  const { session } = require('electron')

  session.defaultSession.on('select-usb-device', (event, details, callback) => {
    event.preventDefault()
    // details.deviceList는 USBDevice 객체의 배열입니다
    const device = details.deviceList.find((d) => d.vendorId === 0x1234)
    callback(device?.deviceId)
  })
})
```

<EasyNote>
`USBDevice`는 USB 포트에 꽂힌 물건의 신분증과 같습니다. 제조사와 모델 번호, 이름, 그리고 어떤 종류의 USB 장치인지를 담고 있습니다.
</EasyNote>

## 쉽게 풀어 설명

앱이 브라우저에게 USB 기기(마이크로컨트롤러, 카드 리더기, 맞춤형 주변기기 등)에 연결해 달라고 요청하면, Electron은 각 후보 장치를 설명하는 `USBDevice` 객체를 전달합니다. 원하는 장치를 알아보려면 `vendorId`와 `productId`를 확인하고, 승인하려면 그 장치의 `deviceId`를 반환하면 됩니다. 버전과 클래스 필드는 하드웨어가 무엇을 말하는지 정확히 알려주지만, 대부분의 앱은 이름과 ID만 있으면 충분합니다.
