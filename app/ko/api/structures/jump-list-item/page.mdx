`JumpListItem`은 Windows 작업 표시줄 점프 목록에 표시되는 항목 하나를 설명합니다. 이 객체들을 배열로 만들어 `JumpListCategory` 안에 담은 뒤 `app.setJumpList(categories)`에 전달합니다. Windows 전용 구조체입니다.

## 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `type` | `string` | _(선택)_ _(Windows)_ `task`(인자와 함께 앱 실행), `separator`(항목 구분선, 기본 `Tasks` 카테고리에서만 유효), `file`(앱에 등록된 핸들러로 파일 열기) 중 하나입니다. |
| `path` | `string` | _(선택)_ _(Windows)_ 열 파일의 경로입니다. `type`이 `file`일 때만 사용됩니다. |
| `program` | `string` | _(선택)_ _(Windows)_ 실행할 프로그램의 경로이며, 보통 `process.execPath`를 사용합니다. `type`이 `task`일 때만 사용됩니다. |
| `args` | `string` | _(선택)_ _(Windows)_ `program` 실행 시 전달되는 명령줄 인자입니다. `type`이 `task`일 때만 사용됩니다. |
| `title` | `string` | _(선택)_ _(Windows)_ 점프 목록에 표시되는 항목 텍스트입니다. `type`이 `task`일 때만 사용됩니다. |
| `description` | `string` | _(선택)_ _(Windows)_ 작업의 툴팁 텍스트입니다(최대 260자). `type`이 `task`일 때만 사용됩니다. |
| `iconPath` | `string` | _(선택)_ _(Windows)_ 아이콘 리소스(`.ico`, `.exe`, `.dll`)의 전체 경로이며, 흔히 `process.execPath`를 사용합니다. |
| `iconIndex` | `number` | _(선택)_ _(Windows)_ 여러 아이콘이 들어 있는 리소스 파일에서 아이콘의 0부터 시작하는 인덱스입니다. 아이콘이 하나뿐이면 `0`을 사용합니다. |
| `workingDirectory` | `string` | _(선택)_ _(Windows)_ 작업이 실행되는 작업 디렉터리입니다. 기본값은 빈 문자열입니다. |

```js
const item = {
  type: 'task',
  program: process.execPath,
  args: '--new-window',
  title: 'New Window',
  description: '새 창을 엽니다',
  iconPath: process.execPath,
  iconIndex: 0,
}
```

<EasyNote>
JumpListItem은 작업 표시줄 아이콘을 오른쪽 클릭했을 때 나오는 바로가기 한 줄, "이 작업 하나만 해줘"라고 적힌 포스트잇 같은 것이라고 생각하시면 됩니다.
</EasyNote>

## 쉽게 풀어 설명

점프 목록은 Windows 작업 표시줄에서 앱 아이콘을 오른쪽 클릭하면 뜨는 작은 메뉴입니다. 각 `JumpListItem`은 그 메뉴의 한 줄에 해당합니다. 앱을 먼저 열지 않고도 "새 문서"나 "최근 파일" 같은 빠른 바로가기를 사용자에게 제공할 때 사용합니다. 대부분의 필드는 `task` 항목에만 의미가 있으며, `separator`와 `file` 항목은 그중 일부만 사용합니다.
