## What you need

<Steps>

### Node.js 20 or newer
Electron runs on top of Node.js. Verify with:

```bash filename="terminal"
node -v   # v20.x or higher is fine
```

<EasyNote>
Node.js is _the engine that runs JavaScript outside of a browser_. The "Main" process of every Electron app is, under the hood, a Node.js program.
</EasyNote>

### npm (ships with Node.js)
The package manager that fetches external code. Confirm with `npm -v`.

### Comfort with the command line
You only need `cd`, `ls`, `mkdir`, `npm install` to get going.

### A code editor
VS Code is the easiest — first-class JS/TS support and a healthy Electron debugging extension scene.

</Steps>

## Optional but recommended: Git

Version control. The tutorial works without it, but any real app will need it before long.

```bash filename="terminal"
git --version
```

## In plain words

It's the same idea as checking your ingredients _before_ you start cooking. To build an Electron app you need four things installed on your computer. The first is Node.js — _the engine that runs JavaScript outside the browser_. The second, npm, comes along with Node.js for free and is basically the _parts store_ for everything else. The third is the terminal, the window where you type commands directly to the computer in plain text. The fourth is a code editor, your writing surface.

The biggest hurdle for beginners is "the terminal looks scary." It really isn't: knowing _five commands_ (`cd`, `ls`, `mkdir`, `npm`, `git`) is enough for the entire tutorial. After a few hours you'll notice that the black window is actually _faster and more precise than the mouse_ for the kind of work you do here.

Each OS asks for _exactly one_ extra thing. On macOS, a single `xcode-select --install`. On Windows, usually nothing right now. On Linux, the distro's basic build tools (`build-essential` or similar). Finish this page and head to [Build your first app](/en/docs/tutorial/first-app) — a real window will be on screen in under five minutes.
