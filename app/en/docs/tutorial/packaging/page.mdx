## What "packaging" even means

So far, `npm start` only runs on your machine. To send the app to a user we need a format their OS knows: macOS wants `.dmg`/`.app`, Windows wants an `.exe` installer, Linux wants `.deb`/`.rpm`/`.AppImage`. **Electron Forge** handles all of this in one command.

<EasyNote>
"Putting a lunchbox around the meal before mailing it to a friend" is exactly the right picture.
</EasyNote>

## Bring in Electron Forge

<Steps>

### Install and convert

```bash filename="terminal"
npm install --save-dev @electron-forge/cli
npx electron-forge import
```

`import` rewrites your existing `package.json` into the shape Forge expects.

### Make installers (for your current OS)

```bash filename="terminal"
npm run make
```

You'll find OS-native installers under `out/`.

</Steps>

## What you get

<Tabs items={['macOS', 'Windows', 'Linux']}>
  <Tabs.Tab>
    A `.app` bundle plus a `.dmg` disk image. Pair with an Apple Developer certificate for _code signing_ (→ [Code Signing](/en/docs/distribution/code-signing)).
  </Tabs.Tab>
  <Tabs.Tab>
    A Squirrel-based `.exe` installer (NSIS is another common option). Hooks straight into the auto-update channel. Pair with an _Authenticode_ certificate via `signtool` for code signing (→ [Code Signing](/en/docs/distribution/code-signing)).
  </Tabs.Tab>
  <Tabs.Tab>
    `.deb` (Debian/Ubuntu), `.rpm` (Fedora), `.AppImage` (distro-independent).
  </Tabs.Tab>
</Tabs>

## Things that usually trip people up

- **Icons** — supply a 256×256 PNG plus `.icns` for macOS and `.ico` for Windows.
- **App name & version** — Forge picks them up from `productName` and `version` in package.json.
- **First build time** — expect 5–10 minutes the first time as native dependencies download.

<Callout type="warning">
Cross-OS building (e.g. making a Windows `.exe` from macOS) _technically_ works but breaks code signing in practice. The real-world pattern is **one VM/runner per OS**.
</Callout>

## In plain words

Up to now the app is a baby bird that only lives _inside your laptop_. To share it, you have to put it in a shape the OS recognizes — `.dmg` for macOS, `.exe` for Windows, `.deb` / `.rpm` / `.AppImage` for Linux. **Electron Forge** is the _automatic packing machine_ that wraps your code into all of these on demand.

Two things surprise newcomers: you need _OS-specific icon files_ (.icns for macOS, .ico for Windows), and the _first build can take 5–10 minutes_. The slowness isn't your code — it's Forge downloading Chromium binaries, per-OS toolchains, and icon converters in the background. After the first run, the next ones finish in a minute or two.

The biggest misconception is "can I build a Windows .exe from my Mac?" Technically yes, but code-signing breaks in cross-OS builds, so real teams don't do that. They build _on a VM or CI runner per OS_. The next page, [Publishing & updates](/en/docs/tutorial/publishing), explains how to actually hand this lunchbox to a friend.
