## 1) Application menu

```js filename="main.js"
const { app, Menu } = require('electron')

const template = [
  {
    label: 'File',
    submenu: [
      { label: 'New', accelerator: 'CmdOrCtrl+N', click: () => console.log('new') },
      { type: 'separator' },
      { role: 'quit' },
    ],
  },
]

Menu.setApplicationMenu(Menu.buildFromTemplate(template))
```

<EasyNote>
`accelerator: 'CmdOrCtrl+N'` is shorthand for _"⌘N on macOS, Ctrl+N everywhere else."_ Electron handles the per-OS mapping for you.
</EasyNote>

## 2) Global shortcut (works even when the app is in the background)

```js filename="main.js"
const { app, globalShortcut } = require('electron')

app.whenReady().then(() => {
  globalShortcut.register('CommandOrControl+Shift+P', () => {
    console.log('Palette!')
  })
})

app.on('will-quit', () => globalShortcut.unregisterAll())
```

## 3) Dialogs

```js filename="main.js"
const { dialog } = require('electron')

dialog.showOpenDialog({
  properties: ['openFile'],
  filters: [{ name: 'Images', extensions: ['png', 'jpg'] }],
}).then(({ filePaths }) => console.log(filePaths))
```

## 4) System notifications

```js filename="renderer or preload"
new Notification('Build finished', { body: 'Took 5.2s' })
```

<EasyNote>
Same shape as the browser Notification API — Electron just routes it to the real OS notification center.
</EasyNote>

## 5) Tray icon

```js filename="main.js"
const { app, Tray, Menu } = require('electron')
const path = require('node:path')

let tray
app.whenReady().then(() => {
  tray = new Tray(path.join(__dirname, 'icon.png'))
  tray.setToolTip('My App')
  tray.setContextMenu(Menu.buildFromTemplate([{ role: 'quit' }]))
})
```

## In plain words

Until now the app is just a window with "Hello World." The moment a user starts to feel "oh, this is a real app" is when it begins behaving like a desktop citizen. A File menu at the top with "New" inside, the ⌘N shortcut working, an Open button popping up the real OS file picker, a small notification when something finishes in the background.

The five tools on this page (menu, global shortcut, dialog, notification, tray) are the basic building blocks of that desktop-ness. The surprising part is how short the code is. A menu is just an _array_, a shortcut is a _string_, a notification is one line. Electron absorbs the per-OS differences so you can write what you _want_, not what each OS wants you to type.

You'll especially like the shortcut format. `'CmdOrCtrl+N'` is a single string that automatically becomes ⌘N on macOS and Ctrl+N on Windows and Linux. Next stop: [Packaging](/en/docs/tutorial/packaging) — that's where _your code_ becomes a real installer your friends can run.
