## SemVer at a glance

`MAJOR.MINOR.PATCH` — e.g. `29.2.0`

| Slot | When it bumps |
|---|---|
| **MAJOR** | Backward-incompatible changes (e.g. `require` → `import` enforcement) |
| **MINOR** | Backward-compatible new features |
| **PATCH** | Bug fixes + security patches |

<EasyNote>
"Can I upgrade without touching my code?" → MINOR/PATCH yes, MAJOR no — read the migration notes first.
</EasyNote>

## Release channels

- **Stable** — what production should track.
- **Beta** — preview of the next major.
- **Alpha** — quick experiments.
- **Nightly** — built daily.

## Support window

Roughly the _current major plus the two previous_ (three lines in total) receive security patches. Older majors may not see backports.

## Recommended cadence

- Check for the latest stable every quarter.
- Don't skip more than one major at a time — migration debt compounds.
