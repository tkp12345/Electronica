## Where to look

Every major releases an updated `BREAKING-CHANGES.md` in the official repo. Walk it as a checklist against your codebase and plan the migration from that.

## Common shapes of breakage

| Shape | Example |
|---|---|
| **API removed** | The `remote` module → use `contextBridge` instead |
| **Default flipped** | `nodeIntegration` defaults to false |
| **Renamed** | Things like `app.runningUnderRosettaTranslation` |
| **Signature** | `dialog.showOpenDialog` became async |

<EasyNote>
A function flipping from sync to Promise sounds tiny — and stacks into a giant when you skip multiple majors. Hop one major at a time.
</EasyNote>

## A safe migration pattern

1. Branch off for the new major.
2. `npm i electron@latest` and **build only**.
3. Fix compile errors one at a time.
4. Run **automated tests**, then manual scenarios.
5. Ship to a beta channel to a small subset of users first.

## Don't ignore deprecation warnings

```js
console.warn('xxx is deprecated, use yyy')
```

That's the announcement that the next major will remove it. Move now and you'll have nothing left to fix later.

## In plain words

A "breaking change" is exactly what it sounds like — code that worked before doesn't work after the upgrade. Imagine a ladder with one rung quietly missing; a major version often comes with a dozen such missing rungs. Electron kindly collects all of them into a single `BREAKING-CHANGES.md`. Migration is essentially walking that file as a checklist against your codebase.

The shapes of breakage are predictable. ① An API removed entirely (the legacy `remote` module → `contextBridge`). ② A default flipped (`nodeIntegration` now false). ③ A function renamed. ④ A function that used to be synchronous now returns a Promise. Number 4 is the nastiest — there's no compile error, just a sudden runtime symptom like "console.log printed `[object Promise]` instead of the value."

The safe migration pattern is short: branch off, run `npm i electron@latest`, and just _build_. Fix compile errors one by one. After automated tests pass, ship the new build to a beta channel of users first. During all of this, _never ignore the `console.warn` deprecation messages_ — those are the announcement that the next major will delete that API. Move now and you'll buy yourself a 30-minute migration next time around.
