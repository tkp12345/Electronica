## A — F

| Term | One-liner |
|---|---|
| **ASAR** | An archive format that bundles many JS files into one for faster startup and simpler packaging |
| **autoUpdater** | The built-in module that lets a running app check for, download, and swap in newer versions |
| **BrowserWindow** | An object that represents a single desktop window; holds the rendered page |
| **Chromium** | The open-source browser engine behind Chrome — Electron's pixel layer |
| **contextBridge** | The safe API for connecting preload and the web page |
| **contextIsolation** | Keeps the page's and preload's JS worlds separate (security) |
| **CSP** | Content-Security-Policy: a note telling the browser which sources of code may run |

## I — N

| Term | One-liner |
|---|---|
| **IPC** | Inter-process communication — the message channel between Main and Renderer |
| **ipcMain / ipcRenderer** | The IPC interfaces on the Main and Renderer side respectively |
| **Main process** | The Node.js "director" process — windows, menus, OS resources |
| **MAS** | Mac App Store — Apple's desktop app marketplace |

## O — Z

| Term | One-liner |
|---|---|
| **Preload script** | A tiny script that runs just before page load; the safe-bridge slot |
| **Renderer process** | One Chromium process per BrowserWindow; the actual UI |
| **Sandbox** | A security container that strips OS privileges from the Renderer |
| **Squirrel** | The auto-update framework Forge uses on Windows and macOS |
| **trayIcon** | The small icon that lives in the system tray / menu bar |
| **webContents** | The object that represents the _web content_ inside a BrowserWindow |

## In plain words

When a topic is new, _the vocabulary itself is the biggest barrier_. Three unfamiliar words in one sentence — "IPC," "preload," "BrowserWindow" — and you stop reading the idea and start trying to memorize labels. This page is a _tiny dictionary_ that lifts that weight. One-line definitions are enough to recognize a word and continue reading.

A few pairs trip people up the most. **Main vs Renderer** is the difference between a _director_ and an _actor_. **IPC vs preload** is the difference between a _walkie-talkie_ and a _mail slot_ — IPC is the transport, preload decides which messages are allowed through. **Sandbox vs Context Isolation** is a _pair of safety layers_ around a single Renderer; the right answer is to leave both on.

Don't try to _memorize_ the table. Use it as a _reference you flip to_ when you need it. If a page has two unfamiliar words in a row, open this page in a new tab, glance at the two words, and jump back. That's faster than any flashcard exercise.
