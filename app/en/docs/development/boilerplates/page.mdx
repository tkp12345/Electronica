## Three to know

| Name | Vibe |
|---|---|
| **electron-quick-start** | Smallest possible starting line, great for learning the shape |
| **electron-forge** + templates (`vite`, `webpack-typescript`) | The official full toolchain |
| **electron-react-boilerplate** | React + TS + Redux/RTK + Jest pre-wired |

## Spin up a Forge template

```bash filename="terminal"
npx create-electron-app@latest my-app --template=vite-typescript
```

<EasyNote>
`--template=` swaps "blank paper" for "graph paper." You also get Vite's fast HMR for free.
</EasyNote>

## How to choose

- _Will this be company code?_ → React boilerplate.
- _Personal learning?_ → quick-start.
- _Demo/throwaway?_ → Fiddle.

## In plain words

When you build a house, you _don't lay every brick yourself_. You start from a _sample home_ that already has foundation, frame, and wiring done, and you decorate it your way. A boilerplate is that sample home.

The three options differ mostly in _size_. **electron-quick-start** is a _studio apartment_ — the smallest, perfect for learning the pattern because you can see exactly which lines are yours. **Electron Forge templates** are a _three-bedroom apartment_ — Webpack/Vite included, plenty of room to actually build something. **electron-react-boilerplate** is a _fully furnished townhouse_ — React, TS, Redux, Jest are all pre-wired and it's a solid starting point for company work.

The most common mistake is grabbing the biggest boilerplate _while you're still learning_. A hundred files in your editor turns into "where do I even look?" and slows you down. The right direction is _start tiny, add parts when you actually need them_.
