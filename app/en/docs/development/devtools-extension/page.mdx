## One-line load

```js filename="main.js"
const { app, session } = require('electron')
const path = require('node:path')

app.whenReady().then(async () => {
  // Point at the unpacked extension folder
  await session.defaultSession.loadExtension(
    path.join(__dirname, 'extensions', 'react-devtools'),
  )
})
```

<EasyNote>
You point at an _unpacked folder_, not the Chrome Web Store. That makes it work offline and in locked-down corporate environments.
</EasyNote>

## Good combos

- React component tree → **React Developer Tools**
- Redux/RTK state → **Redux DevTools**
- Network → the built-in DevTools _Network_ tab

## Don't ship them in production

```js
if (process.env.NODE_ENV === 'development') {
  await session.defaultSession.loadExtension(/* ... */)
}
```

## In plain words

React DevTools, Redux DevTools — the same extensions you already use in Chrome _just work_ inside Electron. That's because the Renderer literally _is_ Chromium. The one twist is that you point at an _unpacked extension folder_ instead of the Chrome Web Store, which is actually a feature: it works offline and in corporate environments where the Web Store is blocked.

The usual stack is the same one most React/Redux developers reach for daily. Component tree and props → **React Developer Tools**. State changes and time travel → **Redux DevTools**. Network → the built-in DevTools _Network_ tab. Get fluent with those three and your debugging speed roughly doubles.

One last thing: _never ship extensions in production builds_. They cost RAM, and they let users peer into your component tree, which isn't great for security or product perception either. A single `NODE_ENV === 'development'` guard around `loadExtension` is enough.
