## In one breath

A sandboxed Renderer runs at the _lowest privilege level_ the OS allows. It cannot call file-system or network APIs directly; it can only _ask_ the Main process via IPC.

<EasyNote>
"The customer can't reach over the counter. They read the menu and _ask the waiter (IPC)_ for everything."
</EasyNote>

## The default is ON

Since Electron 20+, `sandbox` is true on every BrowserWindow by default. Leave it that way.

```js filename="main.js"
new BrowserWindow({
  webPreferences: {
    sandbox: true, // default — fine to write it explicitly
    contextIsolation: true,
  },
})
```

## Reasons people _think_ they need to turn it off (usually a trap)

- "I want Node APIs in my preload" → wrong reason — preload still has access to a curated set of Node modules with sandbox on.
- "An old library calls `require`" → move that library into the Main process instead.

## Cases where turning it off is actually fine

- A fully trusted, fully local design demo with no remote content.
- A throwaway debug window.

Outside of those, keep it on.

## In plain words

Picture a _sandbox pit_ at a playground. Inside the pit you can play however you like, but you can't reach the _flower beds, road, or sports field_ around it. Whatever happens in the sandbox, the damage stays in the sandbox. The Renderer's sandbox is the same idea — even if your page gets hacked, the OS has fenced it in so the damage stays inside that single window.

Since Electron 20, the sandbox is _on by default_. You don't have to flip a switch; you just have to _not turn it off_. Old blog posts sometimes recommend disabling it "so the preload can use Node freely," but that advice is outdated — preload still has a separate, curated way to pull in the Node modules it needs while the sandbox stays on.

There are very few legitimate reasons to disable it: a _fully trusted, fully local_ demo that never loads remote content, or a throwaway debug window. Outside of that, leave it on, and pair it with its sibling [Context Isolation](/en/docs/processes/context-isolation). Together those two options form the spine of Electron's security model.
