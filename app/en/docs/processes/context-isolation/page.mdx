## Why it matters

If preload and the web page shared one JavaScript world, a hostile script on the page could read preload internals and grab IPC handles. **Context isolation** drops a glass wall between them.

<Callout type="info" emoji="🧒">
"Two people in the same room share the desk. Put them in separate rooms and only the _service hatch (contextBridge)_ lets things move between."
</Callout>

## The safe way to expose things: `contextBridge`

```js filename="preload.js"
const { contextBridge, ipcRenderer } = require('electron')

contextBridge.exposeInMainWorld('api', {
  // expose functions, never raw objects
  save: (data) => ipcRenderer.invoke('save', data),
  onUpdate: (callback) => {
    ipcRenderer.on('update', (_, payload) => callback(payload))
  },
})
```

<Callout type="warning">
Don't expose the raw `ipcRenderer` or `fs` modules. Cherry-pick the functions you need.
</Callout>

## A common mistake

```js
// 🚫 wrong: hands over the whole ipcRenderer
contextBridge.exposeInMainWorld('ipc', ipcRenderer)
```

That effectively cancels your security. An attacker can now send any IPC message they like.

```js
// ✅ right: allow-list the channels you actually use
contextBridge.exposeInMainWorld('ipc', {
  invoke: (channel, ...args) => {
    const allow = ['save', 'load', 'export-csv']
    if (!allow.includes(channel)) throw new Error('forbidden')
    return ipcRenderer.invoke(channel, ...args)
  },
})
```
