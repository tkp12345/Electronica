## In one line

Forge bundles Electron + Webpack/Vite + makers (.dmg/.exe/.deb installers) + publishers (GitHub/S3) into _one command chain_.

| Stage | Command |
|---|---|
| Dev | `npm start` |
| Package (no installer yet) | `npm run package` |
| Build installers | `npm run make` |
| Upload too | `npm run publish` |

## Start a new project

```bash filename="terminal"
npx create-electron-app@latest my-app --template=vite-typescript
cd my-app
npm start
```

<EasyNote>
`--template=` swaps "blank paper" for "graph paper" — and you get Vite's fast HMR for free.
</EasyNote>

## Config file

```js filename="forge.config.js"
module.exports = {
  packagerConfig: { asar: true, icon: 'assets/icon' },
  makers: [
    { name: '@electron-forge/maker-squirrel' },   // Windows
    { name: '@electron-forge/maker-zip',  platforms: ['darwin'] },
    { name: '@electron-forge/maker-deb' },         // Debian/Ubuntu
    { name: '@electron-forge/maker-rpm' },         // Fedora
  ],
}
```

## In plain words

A few years ago, shipping an Electron app meant _gathering the tools yourself_ — a build tool, an installer maker, a code-signing tool, an uploader, each with its own config and quirks. First-time releases routinely took weeks. **Electron Forge** is _the whole toolbox in one box_. `npm start` for dev, `npm run make` for installers, `npm run publish` for upload — three commands, one chain.

The biggest perk is the _templates_. A single `npx create-electron-app@latest my-app --template=vite-typescript` gives you a project with Vite + TypeScript + secure defaults already wired up. Instead of starting from blank paper, you start on graph paper.

Everything lives in `forge.config.js`. _packagerConfig_ handles app-level packaging (icon, asar, etc.). _makers_ list what installers to produce (Windows .exe, macOS .zip/.dmg, Linux .deb/.rpm). _publishers_ describe where to upload (GitHub, S3, ...). Get comfortable with those three blocks and you've covered most of what a release pipeline ever needs.
