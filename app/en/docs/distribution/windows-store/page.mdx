## Direct vs Microsoft Store

| Item | Direct (.exe) | Microsoft Store (.msix) |
|---|---|---|
| Install | Squirrel installer | Store or sideload |
| Auto-update | `update-electron-app` | Store handles it |
| Privileges | Plain Windows | UWP manifest |
| Code signing | EV cert recommended | Microsoft signs |

## Build a .msix

```bash filename="terminal"
npx electron-windows-store \
  --input-directory ./out/My-App-win32-x64 \
  --output-directory ./out/store \
  --package-version 1.0.0.0 \
  --package-name MyApp
```

<EasyNote>
The converter takes your existing .exe folder and rewraps it as the .msix the Store recognizes.
</EasyNote>

## Things to watch

- **Win32 vs UWP** — some APIs must be declared in the UWP manifest.
- **4-part version number** (e.g. `1.0.0.0`) — Store rule.
- **App ID** — issued through Partner Center.
