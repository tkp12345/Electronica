## Snap traits

- **Universal Linux** — same `.snap` regardless of distro.
- **Auto-update** — the Snap Store keeps it fresh in the background.
- **Sandbox** (AppArmor-based) — permissions are explicit.
- Four release channels: **Stable / Candidate / Beta / Edge**.

## Forge maker

```js filename="forge.config.js"
{
  name: '@electron-forge/maker-snap',
  config: {
    summary: 'My cool app',
    description: 'Cooler than yours',
  },
}
```

<Callout type="info" emoji="🧒">
".deb is the _local distro shop_; .snap is the _nationwide chain_." Works the same everywhere but is heavier because it ships a container.
</Callout>

## Upload

```bash filename="terminal"
snapcraft login
snapcraft upload my-app_1.0.0_amd64.snap --release=stable
```

## Compare with other options

- `.deb` / `.rpm` — light, distro-native.
- `.AppImage` — no install, single file.
- **Snap** — biggest win is auto-update.
