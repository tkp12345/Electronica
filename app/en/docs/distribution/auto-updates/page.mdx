## The easy path

```js filename="main.js"
const { updateElectronApp } = require('update-electron-app')
updateElectronApp({ repo: 'your-name/your-app' })
```

That one line covers _"on every launch, check GitHub Releases, download a newer version if any, swap it in on next restart."_

<EasyNote>
The same way your browser quietly updates itself — the user never clicks a button.
</EasyNote>

## Under the hood

1. The built-in **autoUpdater** sends the _current version_ to your update server and asks "got anything newer?"
2. If yes, it downloads in the background.
3. Once done, fires `update-downloaded`.
4. The next quit/restart atomically swaps binaries.

## Code signing is mandatory

The new version must carry the _same signature_ as the running app, or the OS rejects the swap. Unsigned builds can't auto-update at all.

<Callout type="info">
This works a little differently per platform. _(macOS)_ the new `.app` must be signed with the **same Team ID / Developer ID** as the running app — Squirrel.Mac verifies it and refuses to swap on mismatch. _(Windows)_ Squirrel.Windows installs the new `.exe` and updates the per-user folder under `%LOCALAPPDATA%\<App>`; Authenticode signing (via `signtool`) is what keeps SmartScreen quiet, but a matching signature isn't strictly enforced for the swap itself.
</Callout>

## Roll your own server

```js
const { autoUpdater } = require('electron')

autoUpdater.setFeedURL({
  url: 'https://updates.example.com/win32/' + app.getVersion(),
})

autoUpdater.checkForUpdates()
```

Pick something like _Squirrel.Mac / Squirrel.Windows / Nuts / Hazel_, or roll your own.

## In plain words

The gap between a good app and one that _truly stays with users for years_ is auto-update. You can't email every user "please download the new version!", so the app has to _check, fetch, and apply_ updates on its own. Electron's `update-electron-app` package does that in a single line: `updateElectronApp({ repo: 'your/repo' })`.

A lot of careful behavior hides behind that one line. On launch, the app _compares the current version to the newest tag on GitHub Releases_; if there's a newer one, it _downloads in the background_; when the user next quits, it _swaps the binary quietly_. From the user's perspective it's just "huh, the design shifted a little today."

If you'd rather run your own update server, `autoUpdater.setFeedURL` is all you need to point at it. Server-side, you can speak the Squirrel.Mac/Windows protocol or stand up an open-source option like Hazel or Nuts in an afternoon. The non-obvious requirement: _code signing must match between old and new versions_. If the signatures diverge, the OS treats the new build as "a different app" and the update silently fails.
