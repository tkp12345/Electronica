`BrowserWindow` is a **class** you create with `new` in the **Main process**. Each instance is one real OS window with a web page (a `WebContents`) living inside it.

```js filename="main.js"
const { BrowserWindow } = require('electron')

const win = new BrowserWindow({
  width: 800,
  height: 600,
  show: false, // create hidden, reveal once the page is painted
})

win.once('ready-to-show', () => win.show())
win.loadFile('index.html')
```

<EasyNote>
`BrowserWindow` is the **picture frame**, and the web page is the **picture inside it**. The frame controls size, position, whether it's maximized or minimized; the page (`win.webContents`) controls what's drawn. Two different jobs, two different objects.
</EasyNote>

## Static methods

Call these on the class itself, not on an instance.

| Method | What it does |
|---|---|
| `BrowserWindow.getAllWindows()` | All open windows as a `BrowserWindow[]` |
| `BrowserWindow.getFocusedWindow()` | The currently focused window, or `null` |
| `BrowserWindow.fromWebContents(webContents)` | The window that owns a given `WebContents`, or `null` |
| `BrowserWindow.fromBrowserView(browserView)` _(Deprecated)_ | The window attached to a `BrowserView`, or `null` |
| `BrowserWindow.fromId(id)` | The window with the given numeric `id`, or `null` |

```js filename="reopen.js"
// macOS: re-create a window when the Dock icon is clicked and none are open
app.on('activate', () => {
  if (BrowserWindow.getAllWindows().length === 0) createWindow()
})
```

## Instance events

Listen with `win.on('event-name', handler)`. Events marked _(macOS)_ / _(Windows)_ / _(Linux)_ only fire on that platform.

### Lifecycle & visibility

| Event | Fires when |
|---|---|
| `page-title-updated` | The document title changed — `preventDefault()` to keep the native title |
| `close` | The window is about to close — `preventDefault()` to cancel |
| `closed` | The window has been closed — drop your reference, it's gone |
| `ready-to-show` | The page has been rendered and can be shown without a visual flash |
| `show` | The window was shown |
| `hide` | The window was hidden |
| `focus` | The window gained focus |
| `blur` | The window lost focus |
| `query-session-end` _(Windows)_ | The OS session is about to end — `preventDefault()` to delay |
| `session-end` _(Windows)_ | The OS session is ending (logout/shutdown) |

```js filename="confirm-close.js"
win.on('close', (event) => {
  if (hasUnsavedWork()) {
    event.preventDefault() // keep the window open
    showSaveDialog()
  }
})

win.on('closed', () => {
  // the window is gone — release the reference so it can be GC'd
  win = null
})
```

<Callout type="warning">
After `closed` fires, the window is destroyed — calling any method on it will throw. Always null out your reference inside the `closed` handler.
</Callout>

### Responsiveness

| Event | Fires when |
|---|---|
| `unresponsive` | The web page stopped responding (long-running script) |
| `responsive` | The web page started responding again |

### Size, position & state

| Event | Fires when |
|---|---|
| `maximize` | The window was maximized |
| `unmaximize` | The window left the maximized state |
| `minimize` | The window was minimized |
| `restore` | The window was restored from minimized |
| `will-resize` _(macOS/Windows)_ | Before a resize — `preventDefault()` to block it |
| `resize` | After the window has resized |
| `resized` _(macOS/Windows)_ | A resize gesture finished |
| `will-move` _(macOS/Windows)_ | Before a move — `preventDefault()` to block it |
| `move` | The window is being moved |
| `moved` _(macOS/Windows)_ | A move gesture finished |
| `always-on-top-changed` | The always-on-top flag toggled |

### Fullscreen

| Event | Fires when |
|---|---|
| `enter-full-screen` | Entered fullscreen |
| `leave-full-screen` | Left fullscreen |
| `enter-html-full-screen` | A page element entered HTML fullscreen |
| `leave-html-full-screen` | A page element left HTML fullscreen |

```js filename="show-when-ready.js"
const win = new BrowserWindow({ show: false })

// Avoids a white flash: only reveal the window once the page is painted
win.once('ready-to-show', () => {
  win.show()
})
```

### Gestures & platform extras

| Event | Fires when |
|---|---|
| `app-command` _(Windows/Linux)_ | An App Command was invoked (e.g. media keys, browser back) |
| `swipe` _(macOS)_ | A 3-finger trackpad swipe |
| `rotate-gesture` _(macOS)_ | A trackpad rotation gesture |
| `sheet-begin` _(macOS)_ | A window sheet opened |
| `sheet-end` _(macOS)_ | A window sheet closed |
| `new-window-for-tab` _(macOS)_ | The native "new tab" button was clicked |
| `system-context-menu` _(Windows/Linux)_ | The system context menu was triggered — `preventDefault()` to suppress |

<EasyNote>
`ready-to-show` is the one event almost every app uses. Create the window with `show: false`, then reveal it on `ready-to-show` — the user never sees an empty white rectangle while the page loads.
</EasyNote>

## Instance methods

### Lifecycle

| Method | What it does |
|---|---|
| `win.destroy()` | Force-close the window **without** firing `close` |

## In plain words

A `BrowserWindow` is exactly what it sounds like: **a window with a web browser inside it**. You make one with `new BrowserWindow(options)`, and from then on you have two things to think about.

1. **The frame** — that's the `BrowserWindow` itself. Where is it (`setBounds`)? How big (`setSize`)? Is it maximized, minimized, fullscreen, always-on-top? Can the user resize or close it? Every one of those is a method or property on `win`.
2. **The picture inside** — that's `win.webContents`, your actual HTML/CSS/JS page. You load it with `loadFile` or `loadURL`, and everything about _what's drawn_ lives there.

The pattern you'll use in almost every app: create the window hidden (`show: false`), load your page, and reveal it on **`ready-to-show`** so the user never sees a white flash. Then handle **`close`** if you have unsaved work, and null out your reference on **`closed`** so the destroyed window can't be touched again.

Remember three names and the rest is lookup: **`loadFile`** (put a page in), **`ready-to-show`** (reveal it cleanly), and **`win.webContents`** (talk to the page). That's the heart of every Electron window.
