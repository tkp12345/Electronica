The `Transaction` object describes one in-app purchase transaction from the macOS App Store. You receive `Transaction` objects from the `inAppPurchase` module's `transactions-updated` event when the state of a purchase changes.

## Fields

| Field | Type | Description |
|---|---|---|
| `transactionIdentifier` | `string` | A string that uniquely identifies a successful payment transaction. |
| `transactionDate` | `string` | The date the transaction was added to the App Store's payment queue. |
| `originalTransactionIdentifier` | `string` | The identifier of the restored transaction by the App Store. |
| `transactionState` | `string` | The current status: `purchasing`, `purchased`, `failed`, `restored`, or `deferred`. |
| `errorCode` | `Integer` | The error code if an error occurred while processing the transaction. |
| `errorMessage` | `string` | The error message if an error occurred while processing the transaction. |
| `payment` | `Object` | The payment details for this transaction (see nested fields below). |
| `payment.productIdentifier` | `string` | The identifier of the purchased product. |
| `payment.quantity` | `Integer` | The quantity purchased. |
| `payment.applicationUsername` | `string` | An opaque identifier for the user's account on your system. |
| `payment.paymentDiscount` | `PaymentDiscount` | _(optional)_ The details of the discount offer to apply to the payment. |

```js
const { inAppPurchase } = require('electron')

inAppPurchase.on('transactions-updated', (event, transactions) => {
  for (const transaction of transactions) {
    if (transaction.transactionState === 'purchased') {
      // Deliver the product, then finish the transaction.
      inAppPurchase.finishTransactionByDate(transaction.transactionDate)
    }
  }
})
```

<EasyNote>
A Transaction is like a receipt for an in-app purchase — it shows what was bought, when, who bought it, and whether the charge went through.
</EasyNote>

## In plain words

When someone buys something inside your macOS app through the App Store, Electron hands you a `Transaction` to tell you how that purchase is going. You check `transactionState` to see if it succeeded, failed, or is still pending, read the `payment` to know which product and how many, and use the identifiers to deliver the goods and mark the transaction complete.
